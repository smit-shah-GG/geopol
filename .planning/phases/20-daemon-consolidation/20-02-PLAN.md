Output: Modified app.py, admin_service.py, admin.py schema, admin routes -- all wired to APScheduler.
</objective>

     ```

     - Call `init_shared_deps()` to create the shared dependency container
     - Call `scheduler = create_scheduler()` to build the AsyncIOScheduler
     - Create `failure_tracker = JobFailureTracker(scheduler)`
     - Call `register_all_jobs(scheduler, failure_tracker)`
     - Call `scheduler.start()`

     - Log: `"APScheduler started with {N} jobs"`

  3. **Delete the Polymarket loop entirely:**
     - Remove the `polymarket_task` creation (`asyncio.create_task(_polymarket_loop(settings))`)
     - Remove the `polymarket_task` cancellation in shutdown
     - Delete the entire `_polymarket_loop()` function (lines 126-197)
     - APScheduler now handles Polymarket via the `polymarket` job registered in registry.py

  4. **In shutdown section** (before Redis/DB close):
     - Replace Polymarket task cancellation with: `await shutdown_scheduler(app.state.scheduler, timeout=30.0)`
     - Log: `"APScheduler shut down"`
     - Then close Redis, then close DB (order matters -- scheduler might still be using DB)

  5. **Keep everything else unchanged:** logging setup, DB init, Redis init, dev API key seeding, create_app factory.

    1. Logging
    2. Ring buffer

    4. Redis
    5. Dev API key
    6. Shared deps (init_shared_deps -- needs DB to be ready)
    7. Scheduler create + register + start

  The lifespan shutdown order must be:
    1. Scheduler shutdown (with 30s timeout)
    2. Redis close
    3. DB close

     last_duration: float | None = None       # seconds, from failure tracker

     consecutive_failures: int = 0            # from failure tracker

     ```
     Keep all existing fields (name, daemon_type, status, last_run, next_run, success_count, fail_count).

  2. **Rewire `AdminService` in `src/api/services/admin_service.py`:**

     a. **Add dependency on failure_tracker:** The service needs both the DB session AND the failure tracker. Add a class-level or constructor param for the failure tracker. Since the service is created per-request via FastAPI Depends, the failure tracker must be passed in. Update constructor:
        ```python
        def __init__(self, db: AsyncSession, failure_tracker: JobFailureTracker | None = None) -> None:
        ```
        The failure_tracker is optional for backward compatibility (tests that don't have a scheduler).

     b. **Rewrite `get_processes()`:**
        - Import `get_scheduler` from `src.scheduler.core`
        - Try to get the scheduler; if not initialized (RuntimeError), fall back to the existing DB-only logic
        - When scheduler is available: iterate `scheduler.get_jobs()`, merge APScheduler live state with IngestRun DB aggregates:
          - `status`: "paused" if `job.next_run_time is None`, "scheduled" if pending, "running" if the job wrapper is currently executing (check failure tracker last_run_time)
          - `next_run`: from `job.next_run_time`
          - `last_run`, `success_count`, `fail_count`: from IngestRun DB (existing query)
          - `last_duration`, `last_error`, `consecutive_failures`: from `failure_tracker.get_job_stats(job.id)`
          - `paused`: `job.next_run_time is None`
        - Map APScheduler job IDs to daemon_types. The registry uses IDs matching daemon types: `gdelt_poller` -> `gdelt`, `rss_tier1` -> `rss`, etc. Create a mapping dict.

     c. **Rewrite `trigger_job()`:**
        - Delete the 501 stub entirely
        - Import `get_scheduler`, `datetime`, `timezone`
        - Get scheduler, get job by constructing the job_id from daemon_type (e.g., `gdelt` -> `gdelt_poller`). Use a `_DAEMON_TO_JOB_ID` mapping dict.
        - Call `scheduler.modify_job(job_id, next_run_time=datetime.now(timezone.utc))`
        - If job not found: raise HTTPException(404)

     d. **Add `pause_job(daemon_type)` method:**
        - Map daemon_type to job_id(s) via `_DAEMON_TO_JOB_IDS`
        - For each job_id: call `scheduler.pause_job(job_id)`
        - Reset failure count via `failure_tracker.reset_failures(job_id)` if tracker available

     e. **Add `resume_job(daemon_type)` method:**
        - Map daemon_type to job_id(s) via `_DAEMON_TO_JOB_IDS`
        - For each job_id: call `scheduler.resume_job(job_id)`
        - Reset failure count via `failure_tracker.reset_failures(job_id)` if tracker available

     f. **Add `reschedule_job(daemon_type, new_interval_seconds)` method:**
        - Map daemon_type to job_id
        - Call `scheduler.reschedule_job(job_id, trigger=IntervalTrigger(seconds=new_interval_seconds))`
        - Persist to system_config table: `{job_id}_schedule` -> `{"v": new_interval_seconds}`

     g. **Update `_DAEMON_NAMES` dict** to include all 9 APScheduler job IDs:

            "gdelt": ["gdelt_poller"],
            "rss": ["rss_tier1", "rss_tier2", "rss_prune"],
            "pipeline": ["daily_pipeline"],
            "polymarket": ["polymarket"],
            "acled": ["acled_poller"],
            "advisory": ["advisory_poller"],
            "tkg": ["tkg_retrain"],
        }

        Note: RSS has 3 APScheduler jobs but is one daemon_type in the admin UI. The process table should show RSS as one row with aggregated state from all 3 sub-jobs.
