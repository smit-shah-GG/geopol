  <action>
  1. Install APScheduler: `uv add "APScheduler==3.11.2"`

  2. Create `src/scheduler/__init__.py` -- export `create_scheduler`, `get_scheduler`, `shutdown_scheduler`, `register_all_jobs`, `init_shared_deps`.

  3. Create `src/scheduler/core.py`:
     - Module-level `_scheduler: AsyncIOScheduler | None = None`
     - `create_scheduler()` -> creates AsyncIOScheduler with:
       - executors: `"default"` = AsyncIOExecutor(), `"processpool"` = APScheduler's ProcessPoolExecutor(max_workers=1)
       - job_defaults: `coalesce=True`, `max_instances=1`, `misfire_grace_time=900`
     - `get_scheduler()` -> returns singleton, raises RuntimeError if not initialized
     - `shutdown_scheduler(scheduler, timeout=30.0)` -> async function that:
       a. Calls `scheduler.pause()` to prevent new job starts
       b. Uses `asyncio.wait_for(asyncio.to_thread(scheduler.shutdown, wait=True), timeout=timeout)`
       c. On TimeoutError: logs warning, calls `scheduler.shutdown(wait=False)`

  4. Create `src/scheduler/dependencies.py`:
     - `SharedDeps` dataclass holding: `settings: Settings`, `event_storage: EventStorage`, `graph: TemporalKnowledgeGraph`, `gdelt_poller: GDELTPoller` (singleton for `_last_url` state persistence across cycles -- see Research pitfall 2)
     - `init_shared_deps()` -> creates singleton, initializes EventStorage and TKG graph
     - `get_shared_deps()` -> returns singleton, raises RuntimeError if not initialized
     - The GDELT poller must be a singleton so `_last_url` persists between poll cycles. Construct it once here with the shared EventStorage and TKG graph.

  5. Create `src/scheduler/job_wrappers.py`:
     - Module-level `_heavy_job_lock = asyncio.Lock()` for heavy job mutual exclusion
     - Module-level `_process_executor: ProcessPoolExecutor | None = None` (stdlib concurrent.futures, NOT APScheduler's wrapper) -- initialized on first use with `max_workers=1`

     Light job wrappers (all async, use default AsyncIOExecutor):
     - `async def gdelt_poll_cycle()` -> gets shared deps, calls `deps.gdelt_poller._poll_once()` (uses singleton poller)
     - `async def rss_poll_tier1()` -> creates RSSDaemon with DaemonConfig from settings, calls `poll_feeds(TIER_1_FEEDS)`, records IngestRun
     - `async def rss_poll_tier2()` -> same as tier1 but with TIER_2_FEEDS
     - `async def rss_prune()` -> creates RSSDaemon, calls `_maybe_prune()`
     - `async def acled_poll_cycle()` -> creates ACLEDPoller from shared deps, calls `_poll_once()`
     - `async def advisory_poll_cycle()` -> creates AdvisoryPoller from settings, calls `_poll_once()`

     Heavy job wrappers (all async, acquire `_heavy_job_lock`, dispatch to ProcessPoolExecutor):
     - `async def heavy_daily_pipeline()` -> acquire lock, then `await loop.run_in_executor(_get_process_executor(), run_daily_pipeline)`
     - `async def heavy_polymarket_cycle()` -> acquire lock, dispatch to `run_polymarket_cycle`
     - `async def heavy_tkg_retrain()` -> acquire lock, dispatch to `run_tkg_retrain`

     Helper: `_get_process_executor()` -> lazily creates `ProcessPoolExecutor(max_workers=1)` singleton

     CRITICAL: Each wrapper must be wrapped in try/except that catches all exceptions, logs them via `logger.exception()`, and re-raises. This prevents APScheduler's traceback retention memory leak (Research pitfall 3). Clear traceback with `sys.exc_clear()` pattern is NOT needed if re-raising -- the listener handles tracking.

     CRITICAL: Do NOT call `asyncio.run()` inside async wrappers. Light jobs use `await` directly. Heavy jobs use `loop.run_in_executor()` to dispatch to the sync functions in heavy_runner.py.

  6. Create `src/scheduler/heavy_runner.py`:
     - All functions MUST be module-level (pickleable by ProcessPoolExecutor)
     - All imports MUST be inside the function body (subprocess worker has fresh Python env)

     **IMPORTANT:** `scripts/` is NOT a Python package (no __init__.py, not in pyproject.toml packages). Do NOT attempt `import scripts.daily_forecast` or `import scripts.retrain_tkg` -- they will raise ModuleNotFoundError. Use `subprocess.run()` to invoke them as CLI scripts, which is exactly how systemd/cron would call them.

     Similarly, `retrain_tkg.main()` calls `argparse.ArgumentParser.parse_args()` which reads `sys.argv`. Calling `main()` directly would lose the `--force` flag unless you manipulate sys.argv. `subprocess.run()` avoids this entirely by passing args naturally on the command line.

     `def run_daily_pipeline() -> int`:
       - Uses `subprocess.run(["uv", "run", "python", "scripts/daily_forecast.py"], capture_output=True, text=True, timeout=3600)`
       - Logs stdout/stderr via logger
       - Returns `result.returncode`
       - On `subprocess.TimeoutExpired`: logs error, returns 1

     `def run_polymarket_cycle() -> int`:
       - This is the ONLY heavy runner function that uses in-process imports (Polymarket code lives in `src/` which IS a proper package)
       - Imports:
         - `import asyncio, aiohttp, logging`
         - `from src.settings import get_settings`
         - `from src.db.postgres import async_session_factory`
         - `from src.forecasting.gemini_client import GeminiClient`
         - `from src.polymarket.auto_forecaster import PolymarketAutoForecaster`
         - `from src.polymarket.client import PolymarketClient`
         - `from src.polymarket.comparison import PolymarketComparisonService`
         - `from src.polymarket.matcher import PolymarketMatcher`
       - Defines an inner `async def _cycle()` that replicates app.py's `_polymarket_loop` logic for ONE iteration (no while-loop, no sleep):
         a. `settings = get_settings()`
         b. `gemini_client = GeminiClient()`
         c. `matcher = PolymarketMatcher(gemini_client, match_threshold=settings.polymarket_match_threshold)`
         d. `async with aiohttp.ClientSession() as http_session:`
         e.   `pm_client = PolymarketClient(session=http_session)`
         f.   `service = PolymarketComparisonService(async_session_factory=async_session_factory, polymarket_client=pm_client, matcher=matcher, settings=settings)`
         g.   `result = await service.run_matching_cycle()`
         h.   `await service.capture_snapshots()`
         i.   `auto_forecaster = PolymarketAutoForecaster(async_session_factory=async_session_factory, gemini_client=gemini_client, settings=settings)`
         j.   `geo_events = await pm_client.fetch_geopolitical_markets()`
         k.   `auto_result = await auto_forecaster.run(geo_events, tracked_ids=set())`
         l.   Re-forecast if not already done today: `from datetime import date; today_str = date.today().isoformat()` -- use a module-level `_last_reforecast_date` to track at-most-once-daily
         m.   `reforecast_result = await auto_forecaster.reforecast_active()`
       - Calls `asyncio.run(_cycle())`
       - Returns 0 on success, 1 on exception (logged)

     `def run_tkg_retrain() -> int`:
       - Uses `subprocess.run(["uv", "run", "python", "scripts/retrain_tkg.py", "--force"], capture_output=True, text=True, timeout=7200)`
       - Logs stdout/stderr via logger
       - Returns `result.returncode`
       - On `subprocess.TimeoutExpired`: logs error, returns 1

  <files>

  <action>
  1. Create `src/scheduler/retry.py`:
     - `JobFailureTracker` class:
       - Constructor: `__init__(self, scheduler, max_consecutive_failures: int = 5)`
       - State: `_consecutive_failures: dict[str, int]`, `_last_error: dict[str, str]`, `_last_duration: dict[str, float]`, `_last_run_time: dict[str, datetime]`
       - `on_job_event(event)` -- APScheduler listener callback:
         a. On `EVENT_JOB_EXECUTED`: reset consecutive failures for job_id, record duration and last_run_time
         b. On `EVENT_JOB_ERROR`: increment consecutive failures, store error message (str(event.exception)), log error. If count >= max_consecutive_failures: call `scheduler.pause_job(event.job_id)`, log warning "Job {id} auto-paused after {N} consecutive failures"
         c. On `EVENT_JOB_MISSED`: log warning "Job {id} missed execution"
       - `get_job_stats(job_id: str) -> dict` -- returns consecutive_failures, last_error, last_duration, last_run_time for admin API consumption
       - `get_all_stats() -> dict[str, dict]` -- returns stats for all tracked jobs
       - `reset_failures(job_id: str)` -- manually reset failure count (called on resume)

     - Job dependency graph (simple dict):
       `JOB_DEPENDENCIES: dict[str, list[str]]` = {
         "daily_pipeline": ["gdelt_poller", "rss_tier1"],  # daily pipeline needs fresh GDELT + RSS data
       }
       Other jobs have no upstream dependencies.

     - `check_upstream_health(job_id: str, tracker: JobFailureTracker) -> bool`:
       Returns True if all upstream jobs for this job_id have 0 consecutive failures today.
       If any upstream has failures > 0, logs warning and returns False.

  2. Create `src/scheduler/registry.py`:
     - `register_all_jobs(scheduler, failure_tracker: JobFailureTracker) -> None`:
       a. Get settings via `get_settings()`
       b. Read schedule overrides from system_config table (sync query or skip if DB not available). For each job, check if system_config has a `{job_id}_schedule` override.
       c. Register all 9 jobs with `scheduler.add_job()`:

       Light jobs (default executor = AsyncIOExecutor):
       - `gdelt_poller`: IntervalTrigger(seconds=settings.gdelt_poll_interval), misfire_grace_time=settings.gdelt_poll_interval
       - `rss_tier1`: IntervalTrigger(seconds=settings.rss_poll_interval_tier1), misfire_grace_time=settings.rss_poll_interval_tier1
       - `rss_tier2`: IntervalTrigger(seconds=settings.rss_poll_interval_tier2), misfire_grace_time=settings.rss_poll_interval_tier2
       - `rss_prune`: IntervalTrigger(seconds=86400), misfire_grace_time=7200
       - `acled_poller`: IntervalTrigger(seconds=settings.acled_poll_interval), misfire_grace_time=3600
       - `advisory_poller`: IntervalTrigger(seconds=settings.advisory_poll_interval), misfire_grace_time=3600

       Heavy jobs (default executor -- async wrappers handle ProcessPoolExecutor internally):
       - `daily_pipeline`: CronTrigger(hour=6, minute=0), misfire_grace_time=3600
       - `polymarket`: IntervalTrigger(seconds=settings.polymarket_poll_interval), misfire_grace_time=1800. Only register if `settings.polymarket_enabled`.
       - `tkg_retrain`: CronTrigger(day_of_week="sun", hour=2), misfire_grace_time=7200

       d. Register failure tracker listener: `scheduler.add_listener(failure_tracker.on_job_event, EVENT_JOB_EXECUTED | EVENT_JOB_ERROR | EVENT_JOB_MISSED)`

     - All jobs use `max_instances=1`, `coalesce=True` (inherited from scheduler defaults, but explicit on each add_job for clarity).
     - Each job gets a human-readable `name` kwarg matching the _DAEMON_NAMES dict pattern.
  </action>
