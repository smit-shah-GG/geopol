```python
class ResolvedComparisonDTO(BaseModel):
    """Single resolved or voided Polymarket comparison for the accuracy table."""
    model_config = ConfigDict(from_attributes=True)

    id: int
    polymarket_title: str
    polymarket_event_id: str
    geopol_probability: float | None = None
    polymarket_price: float | None = None
    polymarket_outcome: float | None = None  # 1.0=Yes, 0.0=No, None=voided
    geopol_brier: float | None = None
    polymarket_brier: float | None = None
    winner: str | None = None  # "geopol" | "polymarket" | "draw" | None (voided)
    status: str  # "resolved" | "voided"
    resolved_at: str | None = None
    created_at: str
    country_iso: str | None = None
    category: str | None = None


class AccuracySummary(BaseModel):
    """Aggregate accuracy stats."""
    total_resolved: int = 0
    total_voided: int = 0
    geopol_wins: int = 0
    polymarket_wins: int = 0
    draws: int = 0
    geopol_cumulative_brier: float | None = None
    polymarket_cumulative_brier: float | None = None
    rolling_30d_geopol_brier: float | None = None
    rolling_30d_polymarket_brier: float | None = None
    rolling_30d_count: int = 0


class AccuracyResponse(BaseModel):
    """Full accuracy endpoint response: summary + resolved comparisons."""
    summary: AccuracySummary
    comparisons: list[ResolvedComparisonDTO]
```

**admin_service.py -- Add get_accuracy() method:**

Add a `get_accuracy()` method to the `AdminService` class. This queries:

1. Summary stats from `polymarket_comparisons` table directly (not from `polymarket_accuracy` ledger -- the ledger is for historical trend, the admin panel shows live-computed data):
   - Count of resolved comparisons (status="resolved")
   - Count of voided comparisons (status="voided")
   - AVG of geopol_brier, polymarket_brier for status="resolved"
   - Count where geopol_brier < polymarket_brier (geopol wins)
   - Count where geopol_brier > polymarket_brier (polymarket wins)
   - Draws = total_resolved - geopol_wins - polymarket_wins

2. Rolling 30-day window: same AVG queries but with `resolved_at >= now - 30 days`

3. Resolved + voided comparisons list, ordered by `resolved_at DESC`, with joined prediction data for `country_iso` and `category`:
   ```python
   stmt = (
       select(PolymarketComparison, Prediction.country_iso, Prediction.category)
       .outerjoin(Prediction, Prediction.id == PolymarketComparison.geopol_prediction_id)
       .where(PolymarketComparison.status.in_(["resolved", "voided"]))
       .order_by(PolymarketComparison.resolved_at.desc())
       .limit(200)
   )
   ```

4. For each comparison, compute `winner`:
   - If status == "voided": winner = None
   - If geopol_brier < polymarket_brier: winner = "geopol"
   - If geopol_brier > polymarket_brier: winner = "polymarket"
   - If equal: winner = "draw"

Return an `AccuracyResponse` with summary and comparisons list.

5. **Voided rows:** Add `.voided` class for visual distinction (reduced opacity, strikethrough on question text).

6. **Auto-refresh:** 30-second interval (same as SourceManager).

7. **Empty state:** When no resolved comparisons exist, show a centered message: "No resolved comparisons yet. Accuracy data appears after Polymarket questions are resolved."
