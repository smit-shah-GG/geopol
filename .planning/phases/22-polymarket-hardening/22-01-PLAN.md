**Rewrite `count_today_reforecasts()` (lines 190-212):**

The current implementation uses `Prediction.created_at >= today` as a proxy for reforecast activity. After the bug fix, this returns 0 forever. Rewrite to query `Prediction.reforecasted_at >= today`:

```python
async def count_today_reforecasts(session: AsyncSession) -> int:
    """Count reforecasts done today using the reforecasted_at column."""
    today_start = datetime.now(timezone.utc).replace(
        hour=0, minute=0, second=0, microsecond=0
    )
    stmt = (
        select(func.count())
        .where(
            Prediction.provenance.in_(["polymarket_driven", "polymarket_tracked"]),
            Prediction.reforecasted_at >= today_start,
        )
        .select_from(Prediction)
    )
    result = await session.execute(stmt)
    return result.scalar() or 0
```

This eliminates the fragile subtraction-based approach (`total - new`). Reforecasts are now directly counted.

**Fix `count_today_new_forecasts()` (lines 173-187):**

The current implementation counts ALL predictions with `created_at >= today` and provenance `polymarket_driven`. With the bug fixed, `created_at` is no longer overwritten on reforecast, so this now correctly counts only genuinely new predictions. No logic change needed, but verify the function is correct by reading it.
