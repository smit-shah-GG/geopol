      - `formState: 'input' | 'parsing' | 'confirm'` -- current state (default `'input'`)
      - `parsedResponse: ParsedQuestionResponse | null` -- result from LLM parse
      - `originalQuestion: string` -- the user's original text (for Edit revert)
      - `errorMessage: string | null` -- error display

    - Constructor: Build initial input-state DOM and append to container.

    - **Input state rendering (`renderInput()`):**
      Create and return a form element:
      ```
      form.submission-form
        label.submission-label "What would you like to forecast?"
        textarea.submission-textarea (placeholder: "e.g. Will Iran retaliate against Israel within the next 30 days?", rows: 4)
        .submission-form-footer
          span.submission-hint "Natural language question -- AI will parse country, timeframe, and category"
          button.submission-submit[type=submit] "Analyze Question"
      ```
      Submit handler (`click` or `submit` event):
      1. Prevent default.
      2. Read textarea value. If empty, show inline validation error and return.
      3. Store as `this.originalQuestion`.
      4. Transition to `'parsing'` state: call `this.setState('parsing')`.
      5. Call `forecastClient.submitQuestion(question)`.
      6. On success: store `parsedResponse`, call `this.setState('confirm')`.
      7. On error: show error message, call `this.setState('input')`.

    - **Parsing state rendering (`renderParsing()`):**
      ```
      .submission-parsing
        .submission-parsing-spinner (CSS animation -- pulsing dot or radar sweep)
        span "Analyzing question..."
      ```

      ```
      .submission-confirm
        .submission-confirm-title "Review parsed question"
        .submission-confirm-grid

            .confirm-value (parsedResponse.question)
          .confirm-field
            .confirm-label "Countries"

          .confirm-field
            .confirm-label "Horizon"

          .confirm-field
            .confirm-label "Category"
            .confirm-value (parsedResponse.category)
        .submission-confirm-actions
          button.submission-edit "Edit"
          button.submission-confirm-btn "Confirm & Submit"
      ```

      Confirm handler:
      1. Disable button, show "Submitting..." text.
      2. Call `forecastClient.confirmSubmission(parsedResponse.request_id)`.
      3. On success: dispatch `submission-confirmed` CustomEvent on `window`.
      4. Reset form: `formState = 'input'`, `parsedResponse = null`, `originalQuestion = ''`, call `setState('input')`.
      5. On error: show error, re-enable button.

      Render appropriate state: `renderInput()`, `renderParsing()`, or `renderConfirm()`.
      Append rendered element to container.
      If transitioning to 'input' and `this.originalQuestion` is non-empty (Edit revert): pre-fill textarea.

    - `destroy(): void` -- no listeners on window to clean up.

      Render same as pending but with badge text "CONFIRMED".

      When requests array is empty:
