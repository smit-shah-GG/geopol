    Class `LayerPillBar`:
    - Constructor takes `deckMap: DeckGLMap` reference.
    - Internal state: `states: Record<LayerId, boolean>` initialized from deckMap's current layer visibility (call `deckMap.getLayerVisible()` for each layer).
    - Layer display config (constant array):
      ```typescript
      const LAYER_PILLS: [LayerId, string][] = [

        ['KnowledgeGraphArcs', 'Arcs'],

        ['ScenarioZones', 'Scenarios'],

      ```
    - `buildBar(): HTMLElement` -- creates a div with class `layer-pill-bar` containing one button per layer. Each button has class `layer-pill`, `data-layer` attribute, and toggles `active` class based on state.
    - `toggle(layerId: LayerId): void` -- flips the state, updates the pill's `active` class, calls `this.map.setLayerVisible(layerId, newState)`.
    - `getElement(): HTMLElement` -- returns the bar container.
    - `destroy(): void` -- no cleanup needed (listeners are on own DOM).

    Import `DeckGLMap, LAYER_IDS, type LayerId` from `@/components/DeckGLMap`.

    **1c. Create `frontend/src/components/GlobeDrillDown.ts`:**

    Slide-in panel from the right edge showing country-specific forecast data.

    Class `GlobeDrillDown`:
    - Private state:
      - `panel: HTMLElement` -- the slide-in container (class `globe-drilldown`)
      - `content: HTMLElement` -- inner scrollable content area
      - `headerName: HTMLElement` -- country name display
      - `headerRisk: HTMLElement` -- risk score + trend arrow
      - `currentIso: string | null` -- currently displayed country
      - `requestToken: number = 0` -- incrementing counter for race condition prevention
      - `expandedIds: Set<string>` -- expanded card state (shared with expandable-card)
      - `cardElements: Map<string, HTMLElement>` -- for potential diff updates

    - Constructor: Build the DOM structure:
      ```
      .globe-drilldown
        .drilldown-header
          .drilldown-header-left
            .drilldown-country-name (country name text)
            .drilldown-risk-score (risk score + trend arrow)
          button.drilldown-close (x button)
        .drilldown-content
          .drilldown-section.drilldown-forecasts (forecast cards go here)
          .drilldown-section.drilldown-sparkline (GDELT sparkline placeholder)
      ```
      Close button click -> `this.close()`.
      Panel click -> `e.stopPropagation()` to prevent globe interaction underneath.

    - `async open(iso: string): Promise<void>`:
      1. Set `this.currentIso = iso`.
      2. Increment `this.requestToken`.
      3. Store token: `const token = this.requestToken`.
      4. Add `active` class to panel (triggers CSS slide-in animation).
      5. Show loading state in content area.
      6. Fetch data in parallel: `forecastClient.getForecastsByCountry(iso)` and `forecastClient.getCountryRisk(iso)`.
      7. **Guard**: if `token !== this.requestToken`, return (user clicked another country).
      8. Update header: country name from `countryGeometry.getNameByIso(iso)`, risk score from CountryRiskSummary.
      9. Render forecast list using `buildExpandableCard()` from expandable-card.ts.
      10. Render risk score section: display current risk_score as 0-100 with trend arrow (rising arrow-up, falling arrow-down, stable dash).
      11. Render sparkline placeholder: "Event data -- Phase 17" with flat line placeholder.

    - `close(): void`:
      Remove `active` class. Set `currentIso = null`. Clear `expandedIds`.

    - `isOpen(): boolean`:
      Return `this.panel.classList.contains('active')`.

    - `getElement(): HTMLElement`:
      Return `this.panel`.

    - `destroy(): void`:
      Close panel, remove DOM.

    Forecast card rendering: Use `buildExpandableCard(f, { expandedIds: this.expandedIds, onToggle: (id, card) => this.toggleCard(id, card) })` for each forecast in the paginated response. The toggle logic mirrors ForecastPanel's toggleCard: add/remove from expandedIds, add/remove expanded-content DOM.

    Private `toggleCard(id: string, card: HTMLElement)`:
    - If expanded: remove from expandedIds, remove `.expanded-content`, remove `.expanded` class.
    - If collapsed: add to expandedIds, add `.expanded` class, append `buildExpandedContent(f)` (parse forecast from `card.dataset.forecastData`).

    At bottom of forecast list, add a "View Details" link that dispatches `country-brief-requested` CustomEvent with `{ iso }` (for CountryBriefPage modal if wired).
