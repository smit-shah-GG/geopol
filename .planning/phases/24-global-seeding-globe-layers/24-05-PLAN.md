  <files>

  <action>
1. Update `src/api/schemas/country.py` -- add dual-score fields:
   - `baseline_risk: float` -- Always present, 0-100. Field(..., ge=0.0, le=100.0)
   - `forecast_risk: float | None` -- Only when active predictions exist. Default None.
   - `blended_risk: float` -- COALESCE(0.7*forecast + 0.3*baseline, baseline). Field(..., ge=0.0, le=100.0)
   - `risk_score: float` -- Backward compat alias for blended_risk. Keep existing field, just compute differently.
   - `top_forecast: str | None` -- NOW NULLABLE (was required). Baseline-only countries have no forecast.
   - `top_probability: float | None` -- NOW NULLABLE (was required).
   - `disputed: bool = False` -- For XK, TW, PS, EH
   - Keep all existing fields (forecast_count, trend, last_updated)

2. Rewrite `src/api/routes/v1/countries.py` -- merge baseline + forecast:
   - The current approach uses a single CTE query against predictions table. This must change to:
     a. Query `baseline_country_risk` table for ALL countries (simple SELECT)
     b. Query predictions table for countries WITH active forecasts (keep existing CTE logic but extract risk_score and top_forecast)
     c. Merge: for each country in baseline results, check if forecast data exists
        - If forecast exists: blended_risk = 0.7 * forecast_risk + 0.3 * baseline_risk
        - If no forecast: blended_risk = baseline_risk, forecast_risk = None
     d. Set risk_score = blended_risk (backward compat)
   - Implementation approach: two queries, Python-side merge. This is cleaner than a massive SQL JOIN because baseline_country_risk is a simple table scan (~195 rows) and the existing forecast CTE is well-tested.
   - The list_countries() endpoint should return ALL countries from baseline_country_risk, not just those with forecasts. This is the key behavior change: globe goes from ~8-15 countries to ~195.
   - The get_country_risk() single-country endpoint should NOT return 404 for countries without forecasts. If baseline_country_risk has the country, return it with forecast_risk=None.
   - Preserve cache layer (same cache keys, same TTL).
   - Keep fixture fallback for dev mode (update _FIXTURE_COUNTRIES to include baseline_risk/blended_risk fields).

3. The `_row_to_dto()` mapping function must be updated to handle the new schema.
  </action>

      - Query `country_arcs` table: SELECT source_iso, target_iso, event_count, avg_goldstein FROM country_arcs

      - Include `computed_at` from the first row
      - Return empty list if no data

   c. `GET /globe/deltas` -> list of significant risk changes:
