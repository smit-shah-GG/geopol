## Task 1: Remove GeoJSON ring winding reversal

**File:** `frontend/src/components/GlobeMap.ts`

**Problem:** `getReversedFeature()` reverses GeoJSON ring coordinates (CW→CCW), but globe.gl handles winding internally. The reversal corrupts polygon rendering and click detection (UAT Tests 3, 4).

**Changes:**
1. In `flushPolygons()` (line ~849), replace `this.getReversedFeature(feature)` with `feature` directly — pass the original GeoJSON feature to globe.gl without modification. Do this for ALL calls to `getReversedFeature()` in `flushPolygons()` (choropleth polygons AND scenario/delta polygons).

2. Delete the `getReversedFeature()` private method entirely (lines 1120-1156).

3. Delete the `reversedFeatureCache` property declaration (line 182) and its `clear()` call in `destroy()` (line 466).

**Verification:**
- `npx tsc --noEmit` passes
- No references to `getReversedFeature` or `reversedFeatureCache` remain in GlobeMap.ts
</task>

<task id="2" name="fix-marker-iso-extraction" commit_type="fix">
## Task 2: Fix forecast marker ISO extraction

**Files:** `frontend/src/components/GlobeMap.ts`, `frontend/src/components/DeckGLMap.ts`

**Problem:** `updateForecasts()` in both renderers extracts country ISO via `f.calibration.category.length === 2`. But `calibration.category` contains CAMEO strings like `"conflict"`, not ISO codes. Markers have NEVER worked (UAT Test 5).

**Changes in BOTH GlobeMap.ts AND DeckGLMap.ts `updateForecasts()` methods:**

Replace the ISO extraction logic. Instead of using `calibration.category`, extract country ISOs from `scenarios[].entities[]`. The entity array contains mixed strings — country ISOs are 2-character uppercase alpha strings matching `/^[A-Z]{2}$/`.

New extraction logic (apply to both files):
```typescript
updateForecasts(forecasts: ForecastResponse[]): void {
  this.markers = [];
  for (const f of forecasts) {
    // Extract country ISOs from scenario entity lists
    const isos = new Set<string>();
    const collectIsos = (scenarios: ForecastResponse['scenarios']): void => {
      for (const s of scenarios) {
        for (const entity of s.entities) {
          const upper = entity.toUpperCase();
          if (/^[A-Z]{2}$/.test(upper)) {
            isos.add(upper);
          }
        }
        if (s.child_scenarios.length > 0) {
          collectIsos(s.child_scenarios);
        }
      }
    };
    collectIsos(f.scenarios);

    // Create a marker for each unique country ISO
    for (const iso of isos) {
      const centroid = countryGeometry.getCentroid(iso);
      if (!centroid) continue;

      this.markers.push({
        iso,
        // GlobeMap uses lat/lng, DeckGLMap uses position: centroid
        // Adapt the property name to match each file's MarkerDatum interface
        ...(/* lat/lng for GlobeMap, position for DeckGLMap */),
        probability: f.probability,
        question: f.question,
      });
    }
  }
  // GlobeMap: this.scheduleFlush()
  // DeckGLMap: this.rebuildLayers()
}
```

**Key differences between files:**
- **GlobeMap.ts:** MarkerDatum has `lat: number, lng: number`. Use `lat: centroid[1], lng: centroid[0]`.
- **DeckGLMap.ts:** MarkerDatum has `position: [number, number]`. Use `position: centroid`.

**Verification:**
- `npx tsc --noEmit` passes
- `grep -n "calibration.category" frontend/src/components/GlobeMap.ts frontend/src/components/DeckGLMap.ts` returns no matches
</task>

**File 2:** `frontend/src/components/LayerPillBar.ts`

**Problem:** `states` cache is constructed once in the constructor and never resyncs when the view mode changes (UAT Test 10).

**Changes:**

```typescript
/** Resync pill states from controller after view mode change. */
syncFromController(): void {
  for (const [id] of LAYER_PILLS) {
    const visible = this.controller.getLayerVisible(id);
    this.states[id] = visible;
    const pill = this.bar.querySelector(`[data-layer="${id}"]`);
    if (pill) {
      pill.classList.toggle('active', visible);
      pill.setAttribute('aria-pressed', String(visible));
    }
  }
}
```

In constructor, after `this.bar = this.buildBar()`:
```typescript
this.modeChangeHandler = () => this.syncFromController();
window.addEventListener('globe-mode-changed', this.modeChangeHandler);
```

In `destroy()`:
```typescript
if (this.modeChangeHandler) {
  window.removeEventListener('globe-mode-changed', this.modeChangeHandler);
}
```
