    **Key design decisions:**
    - DeckGLMap and GlobeMap are NOT imported directly at top level -- they are passed in or dynamically imported. The MapContainer constructor receives the container element and creates sub-containers for each renderer. The actual class imports happen in globe-screen.ts via dynamic import.
    - Alternatively, MapContainer can accept pre-constructed instances. Pick whichever approach avoids circular imports. The cleanest: MapContainer constructor takes `(container: HTMLElement, DeckGLMapClass: typeof DeckGLMap, GlobeMapClass: typeof GlobeMap)` as a factory pattern.
    - ACTUALLY, the simplest approach: MapContainer creates two child divs, constructs both renderers, and exposes the same 13 public methods. globe-screen.ts dynamically imports MapContainer (which internally imports DeckGLMap and GlobeMap).

    **Constructor:**
    ```typescript
    constructor(container: HTMLElement) {
      this.container = container;
      const pref = localStorage.getItem('geopol-globe-mode') ?? '3d';

      // Create two sub-containers -- one for each renderer
      this.deckContainer = h('div', { style: 'position:absolute;inset:0;' });
      this.globeContainer = h('div', { style: 'position:absolute;inset:0;' });
    ```

    ```typescript
    private applyVisibility(): void {
      if (this.deckContainer) {

      if (this.globeContainer) {
        this.globeContainer.style.display = this.activeMode === '3d' ? 'block' : 'none';

    }
    ```
