<task type="auto">
  <name>Task 2: Create GlobeMap.ts -- 3D globe renderer with DeckGLMap-compatible API</name>
  <files>frontend/src/components/GlobeMap.ts</files>
  <action>
    Create `frontend/src/components/GlobeMap.ts` that wraps globe.gl and exposes the same 13 public methods as DeckGLMap. This is the largest artifact in Phase 27.

    **Read DeckGLMap.ts thoroughly first** to understand every public method signature, then read WM's GlobeMap.ts for implementation patterns (adapt, not copy).

    **Class structure:**

    ```typescript
    import Globe from 'globe.gl';
    import { countryGeometry, normalizeCode } from '@/services/country-geometry';
    import { h } from '@/utils/dom-utils';
    import type { Feature, Geometry } from 'geojson';
    import type { ForecastResponse, CountryRiskSummary } from '@/types/api';
    import type { LayerId } from '@/components/DeckGLMap';
    // Re-export data interfaces from DeckGLMap for shared use
    import type { HexBinDatum, BilateralArcDatum, RiskDeltaDatum } from '@/components/DeckGLMap';
    ```

    **Constructor / initialization:**
    - Accept `container: HTMLElement` (same as DeckGLMap)
    - Create wrapper div with `position: relative; width: 100%; height: 100%; overflow: hidden; background: #000;`
    - Initialize globe.gl instance: `Globe()(wrapperEl)` pattern
    - Set globe dimensions from container size
    - Configure globe: `.globeImageUrl('/textures/earth-topo-bathy.jpg')`, `.backgroundColor('rgba(0,0,0,0)')`, `.showAtmosphere(false)` (we do custom atmosphere)
    - Set `pointOfView` to global default: `{ lat: 20, lng: 0, altitude: 1.8 }`
    - Wire `onGlobeReady` callback to apply atmosphere glow + materials + auto-rotate
    - Set renderer config: `powerPreference: 'high-performance'`, antialias when `devicePixelRatio > 1`
    - Wire ResizeObserver on container (same pattern as DeckGLMap)

    **Atmosphere glow (adapt from WM, use Geopol blue #4080dd):**
    - Access Three.js scene via `globe.scene()`
    - Replace default globe material with MeshStandardMaterial: roughness 0.8, metalness 0.1, emissive #0a1f2e at intensity 0.3
    - Add PointLight at (-10,-10,-10) with color #4080dd, intensity 0.3
    - Add outer glow sphere: SphereGeometry(2.15, 64, 64) + MeshBasicMaterial(color: #4080dd, BackSide, transparent, opacity: 0.15)
    - Add inner glow sphere: SphereGeometry(2.08, 64, 64) + MeshBasicMaterial(color: #3060aa, BackSide, transparent, opacity: 0.1)
    - Dynamic import `three` inside this method (not top-level) to avoid loading Three.js types at module scope

    **Auto-rotate:**
    - Slow orbit when user isn't interacting: set `globe.controls().autoRotate = true`, `autoRotateSpeed = 0.3`
    - Pause on mouse interaction (mousedown/touchstart → `autoRotate = false`)
    - Resume after 120s idle timeout (mouseup/touchend → start timer → `autoRotate = true`)
    - Store timeout handle for cleanup in destroy()

    **Data stores (mirror DeckGLMap internals):**
    - `riskScores: Map<string, number>` -- country ISO -> risk score
    - `markers: MarkerDatum[]` -- forecast marker positions
    - `hexBinData: HexBinDatum[]` -- heatmap hex centers
    - `bilateralArcs: BilateralArcDatum[]` -- global arcs
    - `riskDeltaIsos: Map<string, number>` -- risk delta per country
    - `selectedCountry: string | null`
    - `selectedForecast: ForecastResponse | null`
    - `scenarioIsos: Set<string>`
    - `arcs: { source: [number, number]; target: [number, number]; sourceIso: string; targetIso: string; }[]` -- per-country arcs
    - `layerVisible: Record<LayerId, boolean>` -- layer toggle state (all true default)

    **13 public methods (MUST match DeckGLMap signatures exactly):**

    1. `updateRiskScores(summaries: CountryRiskSummary[]): void` -- cache scores, scheduleFlush
    2. `updateForecasts(forecasts: ForecastResponse[]): void` -- build markers from centroids, scheduleFlush
    3. `setSelectedCountry(iso: string | null): void` -- cache, buildArcsForCountry, fly-to via pointOfView, scheduleFlush
    4. `setSelectedForecast(forecast: ForecastResponse | null): void` -- collect scenarioIsos, scheduleFlush
    5. `updateHeatmapData(data: HexBinDatum[]): void` -- cache, scheduleFlush
    6. `updateArcData(data: BilateralArcDatum[]): void` -- cache, scheduleFlush
    7. `updateRiskDeltas(deltas: RiskDeltaDatum[]): void` -- cache delta map, scheduleFlush
    8. `flyToCountry(iso: string): void` -- `globe.pointOfView({ lat, lng, altitude: 1.2 }, 1200)` animated
    9. `setLayerVisible(layerId: LayerId, visible: boolean): void` -- update state, scheduleFlush
    10. `setLayerDefaults(defaults: Partial<Record<LayerId, boolean>>): void` -- batch update, scheduleFlush
    11. `getLayerVisible(layerId: LayerId): boolean` -- read state
    12. `getMap(): null` -- always returns null (no maplibre in 3D mode)
    13. `destroy(): void` -- globe._destructor(), dispose Three.js objects, clear timers, remove ResizeObserver

    **Debounced flush (WM dual-timer pattern):**
    - `scheduleFlush()`: 100ms trailing debounce + 300ms max wait
    - `flushImmediate()`: rebuild all globe.gl data channels based on current state + layer visibility
    - This is CRITICAL -- globe.gl triggers full rebuild on every data setter. Without debounce, 7 data pushes = 7 rebuilds.

    **flushImmediate() -- layer rendering (5 channels):**

    **Layer 1: Risk Choropleth (`polygonsData`)** -- if `ForecastRiskChoropleth` visible:
    - Get all country features from countryGeometry
    - For each feature: get ISO code, look up risk score, compute cap color (same riskColor function as DeckGLMap but as CSS hex string with ~10-15% brightness bump for 3D)
    - Push as polygonsData with `polygonGeoJsonGeometry`, `polygonCapColor`, `polygonStrokeColor('rgba(80,85,95,0.6)')`, `polygonSideColor('rgba(0,0,0,0)')`, `polygonAltitude(0.002)`
    - IMPORTANT: Check GeoJSON ring winding order. Natural Earth uses clockwise; Three.js/globe.gl expects CCW. If polygons render inverted, implement `getReversedRing()` with feature cache.
    - Wire `onPolygonClick` to dispatch `country-selected` CustomEvent (CRITICAL -- WM's is a NO-OP, Geopol MUST implement this):
      ```typescript
      .onPolygonClick((polygon: any, event: MouseEvent, coords: {lat: number, lng: number}) => {
        const code = normalizeCode(polygon.properties);
        if (!code) return;
        window.dispatchEvent(
          new CustomEvent('country-selected', { detail: { iso: code }, bubbles: true })
        );
      })
      ```

    **Layer 2: Forecast Markers (`htmlElementsData`)** -- if `ActiveForecastMarkers` visible:
    - Map markers to `{ lat, lng, size, color, label }` objects
    - Use `htmlElementsData` with small colored dot + optional label
    - Size proportional to probability, color from riskColor(probability)
    - Position via `htmlLat`, `htmlLng`, `htmlAltitude(0.01)`

    **Layer 3: Knowledge Graph Arcs (`arcsData`)** -- if `KnowledgeGraphArcs` visible:
    - If selectedCountry: use per-country arcs (same buildArcsForCountry logic as DeckGLMap)
    - If no selectedCountry: use bilateralArcs (global view)
    - Map to arcsData with `arcStartLat/Lng`, `arcEndLat/Lng`, `arcColor` (sentiment-based: cooperative=green, conflictual=red), `arcDashLength(0.4)`, `arcDashGap(0.2)`, `arcDashAnimateTime(2000)`, `arcStroke(0.5)`

    **Layer 4: Heatmap (colored point markers)** -- if `GDELTEventHeatmap` visible:
    - Convert H3 hex bin centers to lat/lng using h3-js `cellToLatLng`
    - Render as `pointsData` with `pointLat`, `pointLng`, `pointColor` (heat scale), `pointAltitude(0.005)`, `pointRadius(0.3)`, `pointsMerge(true)` for performance

    **Layer 5: Scenario Zones (`polygonsData` overlay)** -- if `ScenarioZones` visible:
    - If selectedForecast: highlight scenarioIsos countries with accent color
    - If no selectedForecast: use riskDeltaIsos for red/green delta highlights
    - Merge with Layer 1 polygons (both use polygonsData -- must be a single combined array)
    - IMPORTANT: Layers 1 and 5 BOTH use polygonsData. globe.gl has one `polygonsData` channel, not two. Combine the features into one array, with scenario/delta features having higher altitude (0.004) to render above choropleth (0.002).

    **buildArcsForCountry()** -- same logic as DeckGLMap lines 849-885: build arcs from selected country centroid to other forecast marker countries and scenario entity ISOs.

    **Color utilities:**
    - Port DeckGLMap's `riskColor()` but output CSS hex strings instead of RGBA arrays
    - Apply ~10-15% brightness bump: multiply RGB by 1.1-1.15, clamp to 255
    - `riskColorCSS(score: number): string` -- returns `#rrggbb`

    **TypeScript considerations:**
    - globe.gl default export is `Globe` constructor. Types may be incomplete -- use `as any` where necessary for globe.gl internal APIs (scene(), controls(), _destructor())
    - Import `type { LayerId }` and data interfaces from DeckGLMap to ensure API compatibility
    - Do NOT import the DeckGLMap class itself (circular dependency risk)

    **What to AVOID and WHY:**
    - Do NOT use globe.gl's built-in atmosphere (`showAtmosphere(true)`) -- it's a basic white glow. Custom Three.js atmosphere gives us Geopol blue (#4080dd).
    - Do NOT call `globe.polygonsData(...)` directly in each data setter -- that triggers full rebuild. Always go through scheduleFlush().
    - Do NOT import `three` at top level -- dynamic import inside applyAtmosphereGlow() only. Avoids loading 600KB Three.js module at parse time.
    - Do NOT use `globe.globeMaterial()` before onGlobeReady fires -- the internal mesh doesn't exist yet.
    - Do NOT assume country GeoJSON polygons render correctly -- test winding order and add reversal if needed.
