  <action>
    **1. TypeScript interfaces** (`frontend/src/admin/admin-types.ts`):
    Mirror the backend Pydantic DTOs exactly:
    - `ProcessInfo`: name, daemon_type, status, last_run (string|null), next_run (string|null), success_count, fail_count
    - `ConfigEntry`: key, value (unknown), type (string), editable, dangerous, description
    - `LogEntry`: timestamp, severity, module, message
    - `SourceInfo`: name, daemon_type, enabled, healthy, last_run (string|null), events_count, tier (string|null)
    - `AdminSection`: string union type = 'processes' | 'config' | 'logs' | 'sources'

    **2. Admin API client** (`frontend/src/admin/admin-client.ts`):
    - `class AdminClient` takes `adminKey: string` in constructor.
    - Private `fetch<T>(path: string, init?: RequestInit): Promise<T>` method that:
      - Prepends `/api/v1/admin` to path
      - Adds `X-Admin-Key` and `Content-Type: application/json` headers
      - Throws on non-ok response with status + path in error message
      - Parses JSON response
    - Methods: `verify()`, `getProcesses()`, `triggerJob(daemon)`, `getConfig()`, `updateConfig(updates)`, `getLogs(params?)`, `getSources()`, `toggleSource(name, enabled)`.
    - Follow the pattern of `frontend/src/services/forecast-client.ts` but simpler (no circuit breaker, no caching -- admin is low-traffic).

    **3. Auth modal** (`frontend/src/admin/components/AuthModal.ts`):
    - `class AuthModal` with `waitForAuth(): Promise<string>` method.
    - Creates a dimmed overlay (`position: fixed; inset: 0; background: rgba(0,0,0,0.7); z-index: 9999`).
    - Centered modal card with:
      - Title: "ADMIN ACCESS" (monospace, red accent)
      - Password input field (type="password")
      - Submit button
      - Error message area (hidden by default)
    - On submit: create temporary `AdminClient` with entered key, call `verify()`.
      - Success: store key in `sessionStorage.setItem('geopol_admin_key', key)`, resolve promise with key, remove overlay.
      - Failure: show error "Invalid admin key", increment attempt counter.
      - 5 failures: disable input + button for 30s, show countdown.
    - `destroy()` method removes overlay from DOM.
    - Enter key submits the form (keydown listener on input).
    - Use `h()` from `@/utils/dom-utils` for all DOM creation.

    **4. Sidebar nav** (`frontend/src/admin/components/AdminSidebar.ts`):
    - `class AdminSidebar` with constructor taking `onNavigate: (section: AdminSection) => void` callback.
    - Renders a vertical nav with 4 items:
      - "Processes" (icon: a simple unicode char or SVG -- discretionary)
      - "Config"
      - "Logs"
      - "Sources"
    - Active item gets `.active` class (red accent background).
    - `setActive(section: AdminSection)` method updates active state.
    - `getElement(): HTMLElement` returns the sidebar nav element.
    - `destroy()` cleans up.
  </action>
