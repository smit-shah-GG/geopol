    - `emit()` converts `logging.LogRecord` to `LogEntry` and appends.

    **5. Admin Pydantic DTOs** (`src/api/schemas/admin.py`):
    - `ProcessInfo`: name (str), daemon_type (str), status (str: running|success|failed|unknown), last_run (datetime|None), next_run (datetime|None), success_count (int), fail_count (int).
    - `ConfigEntry`: key (str), value (Any), type (str: int|float|str|bool|list), editable (bool), dangerous (bool), description (str).
    - `ConfigUpdate`: updates (dict[str, Any]).
    - `LogEntryDTO`: timestamp (str), severity (str), module (str), message (str).
    - `SourceInfo`: name (str), daemon_type (str), enabled (bool), healthy (bool), last_run (datetime|None), events_count (int), tier (str|None -- for RSS feeds).

    **6. Admin service** (`src/api/services/admin_service.py`):
    - `class AdminService` with async methods, takes `AsyncSession` in constructor.

    - `get_processes() -> list[ProcessInfo]`: Query `ingest_runs` table grouped by `daemon_type`. For each daemon type (gdelt, rss, pipeline, polymarket, acled, advisory):
      - `last_run`: MAX(started_at)
      - `status`: latest row's status (or "unknown" if no rows)
      - `success_count`: COUNT WHERE status='success'
      - `fail_count`: COUNT WHERE status='failed'
      - `next_run`: last_run + poll_interval from Settings (gdelt_poll_interval, rss_poll_interval_tier1, polymarket_poll_interval, acled_poll_interval, advisory_poll_interval). Return None if no last_run.
      - Human-readable names: gdelt -> "GDELT Poller", rss -> "RSS Poller", pipeline -> "Daily Pipeline", polymarket -> "Polymarket Forecaster", acled -> "ACLED Poller", advisory -> "Advisory Poller".

    - `trigger_job(daemon_type: str) -> None`: Spawn a one-shot `asyncio.Task` running the existing ingest/pipeline function. Map daemon_type to function:
      - "gdelt": import and call `src.ingest.gdelt_ingest.run_gdelt_cycle` (or equivalent -- check the actual function name)
      - "polymarket": run a single iteration of the Polymarket matching cycle from app.py
      - For daemon types without a clear one-shot function, log a warning and raise HTTPException(501, "Trigger not yet implemented for {daemon_type}"). Phase 20 (APScheduler) will consolidate all triggers.

    - `get_config() -> list[ConfigEntry]`: Read all settings from RuntimeSettings (system_config table first, Settings defaults as fallback). Mark secrets as non-editable: admin_key, gemini_api_key, database_url, redis_url, smtp_password, acled_email, acled_password. Mark dangerous: gdelt_poll_interval (if < 60), rss_poll_interval_tier1 (if < 60), gemini_daily_budget (if 0), polymarket_enabled (if false). Include descriptions for each setting.

    - `update_config(updates: dict[str, Any]) -> None`: For each key in updates, validate type matches the Settings field type. Reject writes to secret fields. Upsert into system_config table. Invalidate any cached runtime settings.

    - `get_sources() -> list[SourceInfo]`: Query ingest_runs for each source type. Check disabled_feeds from system_config for RSS feeds. Return health = last successful run within 2x poll_interval.

    - `toggle_source(source_name: str, enabled: bool) -> None`: For RSS feeds, update disabled_feeds list in system_config. For other sources (gdelt, acled, advisory), store enabled state in system_config as `{source}_enabled` key.

    **1. Admin API router** (`src/api/routes/v1/admin.py`):
    Create the admin router with `verify_admin` dependency applied to ALL endpoints. Follow the pattern from `src/api/middleware/auth.py` but use Header-based `X-Admin-Key` comparison against `Settings.admin_key`.

    Endpoints:

    - `POST /verify` -> `{"status": "authenticated"}` (lightweight auth check for frontend session validation)
    - `GET /processes` -> `list[ProcessInfo]` (calls `AdminService.get_processes()`)
    - `POST /processes/{daemon_type}/trigger` -> `{"status": "triggered", "daemon_type": ...}` (calls `AdminService.trigger_job()`)
    - `GET /config` -> `list[ConfigEntry]` (calls `AdminService.get_config()`)
    - `PUT /config` -> `{"status": "updated", "keys": [...]}` (calls `AdminService.update_config()`, accepts `ConfigUpdate` body)
    - `GET /logs` -> `list[LogEntryDTO]` (reads from RingBufferHandler via `get_ring_buffer().get_entries()`, accepts optional `severity` and `subsystem` query params)
    - `GET /sources` -> `list[SourceInfo]` (calls `AdminService.get_sources()`)
    - `PUT /sources/{source_name}/toggle` -> `{"status": "toggled", "source": ..., "enabled": ...}` (calls `AdminService.toggle_source()`, accepts `{"enabled": bool}` body)

    The `verify_admin` dependency:
    ```python
    async def verify_admin(
        x_admin_key: str | None = Header(None, alias="X-Admin-Key"),
    ) -> None:
        settings = get_settings()
        if not settings.admin_key:
            raise HTTPException(503, "Admin access not configured -- set ADMIN_KEY env var")
        if x_admin_key != settings.admin_key:
            raise HTTPException(401, "Invalid admin key")
    ```

    Apply `dependencies=[Depends(verify_admin)]` at the router level (not per-endpoint) so ALL admin routes require auth.

    **2. Register admin router** (`src/api/routes/v1/router.py`):
    Add `from src.api.routes.v1.admin import router as admin_router` and `v1_router.include_router(admin_router, prefix="/admin", tags=["admin"])`.
    Place it AFTER the health router (admin is also somewhat "infrastructure" but requires auth).

    **Important:** Admin auth is completely separate from the existing `verify_api_key` system. Admin uses `X-Admin-Key` header, user API uses `X-API-Key` header. They are different key stores (env var vs DB table).
