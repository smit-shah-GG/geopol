    **src/api/routes/v1/events.py:** Create a FastAPI router with `GET ""` (no trailing path) that serves paginated events.

    Query parameters: country (str), start_date (str, YYYY-MM-DD), end_date (str, YYYY-MM-DD), cameo_code (str), actor (str), goldstein_min (float, ge=-10, le=10), goldstein_max (float, ge=-10, le=10), text (str), source (str, 'gdelt' or 'acled'), cursor (str), limit (int, default=50, ge=1, le=200).

    **src/api/routes/v1/articles.py:** Create a FastAPI router with `GET ""` for RSS articles.

    Two modes:
    1. **Keyword mode** (default): Query ChromaDB collection with metadata filters for country, date range, and keyword in content. Use `collection.get()` with `where` clause for metadata filters, then filter by keyword in Python if text search param provided.
    2. **Semantic mode** (`?semantic=true`): Use `collection.query()` with the text param as the query string. Set `n_results=20`. Include `relevance_score` in response.

    Query parameters: country (str), start_date (str), end_date (str), text (str, required for semantic mode), semantic (bool, default false), cursor (str), limit (int, default=20, ge=1, le=100).

    Map ChromaDB results to ArticleDTO. For keyword mode, extract title from metadata, snippet from first 200 chars of document content. The ChromaDB persist dir comes from the RSS daemon config (default `./chroma_db`). Collection name: follow whatever the existing `ArticleIndexer` uses (check `src/ingest/article_processor.py`).

    **src/api/routes/v1/sources.py:** Create a FastAPI router with `GET ""` for data source auto-discovery.

    Query the PostgreSQL `ingest_runs` table to find the last run per `daemon_type`. For each daemon type seen (gdelt, rss, acled, advisory), return a `SourceStatusDTO` with:
    - `name`: daemon_type
    - `healthy`: last run status == "success"
    - `last_update`: completed_at timestamp of last run
    - `events_last_run`: events_new from last run
    - `detail`: formatted status string (e.g., "success: 47 new events")

    Also check for daemon types that SHOULD exist but have no runs yet (return healthy=false, detail="Never run"). The known daemon types are: "gdelt", "rss", "acled", "advisory".

    Use `AsyncSession` from `src/db/postgres.py` (existing pattern from other routes).

    **src/api/routes/v1/advisories.py:** Create a FastAPI router with `GET ""` for government travel advisories.

    This endpoint reads cached advisory data. The advisory poller (Task 2 of Plan 03) stores advisories in memory (module-level cache dict) with a TTL. The endpoint reads from this cache. If cache is empty/expired, return empty list with appropriate message.

    Query parameters: country (str, optional -- filter by country_iso).

    Design decision: advisories are fetched daily and change rarely. An in-memory cache (populated by the advisory poller daemon or on-demand if cache is cold) is sufficient. No database table needed -- advisories are transient display data. Store as a module-level `_advisory_cache: dict` with `_cache_updated_at: float` timestamp.

    Actually, to avoid coupling between the route module and the poller daemon, use a shared advisory store. Create a simple AdvisoryStore class in the advisories route file:
    ```python
    class AdvisoryStore:
        """In-memory cache for government travel advisories, populated by poller."""
        _advisories: list[dict] = []
        _updated_at: float = 0.0

        @classmethod
        def update(cls, advisories: list[dict]) -> None: ...

        @classmethod
        def get(cls, country: str | None = None) -> list[AdvisoryDTO]: ...

        @classmethod
        def last_updated(cls) -> float: ...
    ```
    Export `AdvisoryStore` so the poller can call `AdvisoryStore.update()`.

  <action>
    Create `src/ingest/acled_poller.py` following the established daemon pattern from `gdelt_poller.py`.

    **ACLEDClient class:**
    - OAuth2 token management: acquire via POST to `https://api.acleddata.com/acled/read` (note: ACLED uses email+key authentication, not OAuth2 -- see research). Actually, per ACLED docs, authentication is via `key` and `email` query parameters, not OAuth2 tokens. The research doc's OAuth2 pattern may be inaccurate. Use the simpler approach: pass `key=ACLED_API_KEY` and `email=ACLED_EMAIL` as query parameters.
    - Fetch events since a given date for the three armed conflict types: Battles, Explosions/Remote violence, Violence against civilians.
    - Paginate with `page=N` parameter (5000 rows per page).
    - Timeout: 60 seconds per request.

    **ACLED-to-Event mapping** (from RESEARCH.md):
    - `gdelt_id`: prefix with "ACLED-" to avoid collision (e.g., "ACLED-AFG12345")
    - `content_hash`: SHA-256 of `"acled|{event_id_cnty}"`
    - `event_date`: from ACLED `event_date` field
    - `time_window`: same as event_date (day granularity)
    - `actor1_code`: ACLED `actor1`
    - `actor2_code`: ACLED `actor2`
    - `event_code`: map event_type to CAMEO -- Battles->"19", Explosions->"18", Violence against civilians->"20"
    - `quad_class`: always 4 (Material Conflict)
    - `goldstein_scale`: -10.0 if fatalities > 0, else -8.0
    - `title`: ACLED `notes` field, truncated to 200 chars
    - `country_iso`: ACLED `iso` field (already ISO 3166-1 alpha-2 -- verify; ACLED uses ISO 3166-1 numeric, may need mapping)
    - `source`: "acled"

    **ACLEDPoller class:**
    Follow GDELTPoller structure:
    - Constructor: takes EventStorage, optional poll_interval (default from settings.acled_poll_interval).
    - `_shutdown` asyncio.Event for graceful SIGTERM.
    - `run()` async method: main loop with backoff (base=300s for daily source).
    - `_poll_once()`: fetch events since last successful run date (stored as instance var). Map to Event objects. Insert via `asyncio.to_thread(self.event_storage.insert_events, events)`. Record IngestRun to PostgreSQL with daemon_type="acled".
    - Backoff on failure, reset on success.
    - Log stats: events fetched, new (inserted), duplicates.

    **ACLED country code handling:** ACLED `iso` field is ISO 3166-1 numeric (e.g., 4 for Afghanistan). Need to convert to alpha-2 (e.g., "AF"). Include a numeric-to-alpha2 mapping dict for the ~40-50 countries where ACLED has active conflict data. Alternatively, ACLED also provides `iso3` field (3-letter). Use `iso3` and convert to alpha-2 with a small mapping, or look up from `pycountry` if available. Check if pycountry is in deps -- if not, use a static mapping dict. Keep it simple: ACLED provides `country` field too; use a static dict of ACLED country names to ISO alpha-2 codes for the conflict-affected countries. OR: ACLED provides an `iso` field that is numeric -- convert via a ~250-entry dict. Claude has discretion on exact approach (per CONTEXT.md).

    Handle gracefully:
    - Auth failure: log error, backoff, retry.
    - Empty results: normal for daily poll, log info and skip.
    - Partial page: last page has < 5000 events, stop pagination.
  </action>

  <action>
    Create `src/ingest/advisory_poller.py` with two API clients and a poller daemon.

    **StateDeptClient class:**
    - Fetch all current advisories from `https://cadataapi.state.gov/api/TravelAdvisories`
    - No authentication required.
    - Parse advisory level (1-4) from the title string: "CountryName - Level N: Description"
    - Extract country ISO code from the response (State Dept uses country names -- need name-to-ISO mapping; or check if the API returns ISO codes in a field).
    - Map to advisory dict: source="us_state_dept", country_iso, level, level_description, title, summary, published_at, updated_at, url.

    **FCDOClient class:**
    - Fetch the index from `https://www.gov.uk/api/content/foreign-travel-advice` to get country links.
    - For each country, fetch individual advice page from `https://www.gov.uk/api/content{base_path}`.
    - Extract `alert_status` array from details.
    - Map alert_status to level: avoid_all_travel_to_whole_country=4, avoid_all_travel_to_parts=3, avoid_all_but_essential_travel_to_parts=2, default=1.
    - Country name to ISO mapping needed (FCDO uses country names in URLs like "afghanistan").
    - Rate limit: add a small delay (0.5s) between individual country fetches to avoid hitting GOV.UK rate limits. Use asyncio.Semaphore(5) for concurrent country fetches.

    **Country name to ISO mapping:** Include a static dict of country names (lowercase, no hyphens/spaces normalized) to ISO alpha-2 codes. Cover ~200 countries. This is a one-time mapping that rarely changes. Claude can generate this dict.

    **AdvisoryPoller class:**
    Follow daemon pattern:
    - Constructor: optional poll_interval (default from settings.advisory_poll_interval).
    - `_shutdown` asyncio.Event for graceful SIGTERM.
    - `run()` async: main loop polling daily with backoff.
    - `_poll_once()`:
      1. Fetch US State Dept advisories.
      2. Fetch UK FCDO advisories (with rate limiting).
      3. Merge: for countries with both sources, keep both entries (multiple advisories per country is valid).
      4. Call `AdvisoryStore.update(all_advisories)` to push to the in-memory cache used by the API route.
      5. Record IngestRun with daemon_type="advisory".
    - Error handling: if one source fails, still process the other. Log warnings for individual country fetch failures in FCDO (don't abort the whole batch).

    **EU EEAS: Deliberately excluded.** Per RESEARCH.md, no structured API exists. Only US State Dept and UK FCDO are implemented. Add a comment noting EEAS was evaluated and dropped.

    Import `AdvisoryStore` from `src.api.routes.v1.advisories` to update the shared cache. If this creates a circular import, move AdvisoryStore to a shared module (e.g., `src/ingest/advisory_store.py`).
  </action>
