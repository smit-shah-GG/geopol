    **schema.sql:** Add two new columns to the events table:
    - `country_iso TEXT` -- ISO 3166-1 alpha-2 country code for the event location
    - `source TEXT NOT NULL DEFAULT 'gdelt'` -- discriminator: 'gdelt' or 'acled'

    Add these indexes:
    - `CREATE INDEX IF NOT EXISTS idx_events_country_iso ON events(country_iso);`
    - `CREATE INDEX IF NOT EXISTS idx_events_country_date_id ON events(country_iso, event_date DESC, id DESC);` -- composite for cursor pagination
    - `CREATE INDEX IF NOT EXISTS idx_events_source ON events(source);`

    Note: ALTER TABLE for existing DBs must be handled at init time. Add a migration block to EventStorage.init_database() that checks if columns exist (via PRAGMA table_info) and runs ALTER TABLE + backfill if missing. The backfill query:
    ```sql
    UPDATE events SET country_iso = json_extract(raw_json, '$.ActionGeo_CountryCode')
    WHERE raw_json IS NOT NULL AND country_iso IS NULL;
    ```
    Fall back to Actor1CountryCode if ActionGeo is NULL:
    ```sql
    UPDATE events SET country_iso = json_extract(raw_json, '$.Actor1CountryCode')
    WHERE raw_json IS NOT NULL AND country_iso IS NULL AND json_extract(raw_json, '$.Actor1CountryCode') IS NOT NULL;
    ```
    Log backfill stats (how many rows updated, how many remain NULL).

    **models.py:** Add two fields to the Event dataclass:
    - `country_iso: Optional[str] = None`
    - `source: str = "gdelt"`
    Place them in the "Metadata" section before raw_json.

    **storage.py:** Add a new method `query_events()` that supports the full filter surface:
    ```python
    def query_events(
        self,
        country: str | None = None,
        start_date: str | None = None,
        end_date: str | None = None,
        cameo_code: str | None = None,
        actor: str | None = None,
        goldstein_min: float | None = None,
        goldstein_max: float | None = None,
        text: str | None = None,
        source: str | None = None,
        cursor_id: int | None = None,
        cursor_date: str | None = None,
        limit: int = 50,
    ) -> list[Event]:
    ```
    Build a parameterized SQL query dynamically. Use keyset pagination: `WHERE (event_date, id) < (?, ?)` when cursor provided. Default time range: 30 days when no date params given. Order by `event_date DESC, id DESC`. Return `limit + 1` rows (caller checks has_more by seeing if len > limit).

    Also add `query_events_count(country, start_date, end_date)` returning total count for a country in a date range (used by country brief page).

    Also add `query_top_actors(country, start_date, end_date, limit=20)` that returns actor codes with event counts for entity tab:
    ```sql
    SELECT actor1_code as actor, COUNT(*) as cnt FROM events
    WHERE country_iso = ? AND event_date BETWEEN ? AND ?
    AND actor1_code IS NOT NULL
    GROUP BY actor1_code
    ORDER BY cnt DESC LIMIT ?
    ```
    Union with actor2_code results, deduplicate, sort by count.

    **gdelt_poller.py:** Modify `_gdelt_row_to_event()` to populate `country_iso` from the GDELT CSV row's `ActionGeo_CountryCode` field (already in GDELT_COLUMNS at index 52). Fall back to `Actor1CountryCode` (index 7) if ActionGeo is empty. The Event constructor call should include `country_iso=country_iso`.

    Do NOT change the Event constructor signature ordering -- add `country_iso` and `source` as keyword arguments only (they have defaults).
