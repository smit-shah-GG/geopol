    **GET /countries endpoint -- CTE-based aggregation query:**

    The SQL logic (implement via SQLAlchemy Core expressions, not raw SQL string):

    1. Filter to active predictions: `WHERE country_iso IS NOT NULL AND expires_at > NOW()`

    2. Compute per-country metrics with exponential time decay:
       - `forecast_count`: COUNT(*) per country_iso
       - `avg_probability`: SUM(probability * decay_weight) / SUM(decay_weight) where decay_weight = EXP(-0.693 * seconds_since_creation / (7 * 86400)) -- 7-day half-life
       - `avg_severity`: CAMEO-to-severity mapping via SQL CASE expression:
         - Material conflict (15-20): 1.0
         - Verbal conflict (10-14): 0.6
         - Material cooperation (06-09): 0.3
         - Verbal cooperation (01-05): 0.1
         - NULL/unknown CAMEO: 0.5
       - Weighted: SUM(severity * decay_weight) / SUM(decay_weight)
       - `top_forecast_question`: question from most recent prediction per country (use a correlated subquery or window function)
       - `top_forecast_probability`: probability from that same prediction
       - `last_updated`: MAX(created_at) per country

    3. Composite risk score (0-100):
       ```
       risk_score = MIN(100, (
           MIN(20, forecast_count * 4) +    -- count component: 0-20 points (caps at 5 forecasts)
           avg_probability * 50 +            -- probability component: 0-50 points
           avg_severity * 30                 -- severity component: 0-30 points
       ))
       ```

    4. Trend computation: Compare current risk indicators to the state 7 days ago.
       - Compute a "past score" by filtering predictions to `created_at <= NOW() - 7 days` and `expires_at > NOW() - 7 days`
       - Trend thresholds: if current - past > 5: "rising", if past - current > 5: "falling", else "stable"
       - If no past data exists (new country), default to "stable"

    **Implementation approach:**

    Since the query is complex, use SQLAlchemy `text()` for the main CTE query to keep it readable, then map results to CountryRiskSummary DTOs. The research provides the SQL pattern. Wrap it in a service function or keep it inline in the route handler (inline is fine for a single query).

    Alternatively, build the query with SQLAlchemy Core expressions if you can keep it readable. The key constraint is: **one query, no N+1**. The composite index `ix_predictions_country_created` already exists and covers the main access pattern.

    **Cache the results** with SUMMARY_TTL (1 hour from cache_service). Cache key: `"countries:all"` for the list, `"countries:{iso}"` for single country.

    **GET /countries/{iso_code} endpoint:**

    Same aggregation query but filtered to a single country_iso. If no predictions exist for that country, return 404 (not mock data). Cache per-country results.

    **Error handling:** Do NOT swallow exceptions with bare except. Let SQLAlchemy errors propagate to FastAPI's error handler.

    **Performance notes:**
    - The exponential decay computation happens in SQL, not Python -- no N+1.
    - The `ix_predictions_country_created` composite index covers the GROUP BY + ORDER BY.
    - With ~1000 predictions, this query should complete in <50ms.
