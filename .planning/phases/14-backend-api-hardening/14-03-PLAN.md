  <action>
    Add a new GET endpoint to the forecasts router in src/api/routes/v1/forecasts.py.

    **IMPORTANT:** The search route MUST be defined BEFORE the `/{forecast_id}` route. FastAPI matches routes in definition order, and `/search` would be captured by `/{forecast_id}` if it comes after. Place it right after the `/top` endpoint and before `/country/{iso_code}`.

    **Endpoint definition:**

    ```python
    @router.get(
        "/search",
        response_model=SearchResponse,
        summary="Search forecasts",
        description="Full-text search over forecast questions with optional country and category filters.",
    )
    async def search_forecasts(
        q: str = Query(..., min_length=2, max_length=200, description="Search query text"),
        country: Optional[str] = Query(default=None, description="Filter by country ISO code"),
        category: Optional[str] = Query(default=None, description="Filter by forecast category"),
        limit: int = Query(default=20, ge=1, le=50, description="Maximum results"),
        _client: str = Depends(verify_api_key),
        db: AsyncSession = Depends(get_db),
    ) -> SearchResponse:
    ```

    **Implementation:**

    1. Build the tsquery from the search text:
       ```python
       from sqlalchemy import func
       query_ts = func.plainto_tsquery('english', q)
       ```

    2. Build the SELECT statement:
       ```python
       rank_expr = func.ts_rank(Prediction.question_tsv, query_ts)
       stmt = (
           select(Prediction, rank_expr.label("relevance"))
           .where(Prediction.question_tsv.op('@@')(query_ts))
       )
       ```

    3. Apply optional filters:
       ```python
       if country:
           stmt = stmt.where(Prediction.country_iso == country.upper())
       if category:
           stmt = stmt.where(Prediction.category == category)
       ```

    4. Get total count before pagination (use a subquery or separate count query):
       ```python
       count_stmt = select(func.count()).select_from(stmt.subquery())
       total = (await db.execute(count_stmt)).scalar_one()
       ```

    5. Apply ordering and limit:
       ```python
       stmt = stmt.order_by(rank_expr.desc()).limit(limit)
       ```

    6. Execute and map results:
       ```python
       result = await db.execute(stmt)
       rows = result.all()
       results = []
       for prediction, relevance in rows:
           dto = ForecastService.prediction_to_dto(prediction)
           results.append(SearchResult(forecast=dto, relevance=float(relevance)))
       ```

    7. Return SearchResponse:
       ```python
       return SearchResponse(results=results, total=total, query=q)
       ```

    **Imports to add:**
    - `from src.api.schemas.search import SearchResponse, SearchResult`
    - Ensure `func` is imported from sqlalchemy (may already be via other imports)

    **Error handling:**
    - If `q` is too short or produces an empty tsquery (e.g., all stop words), return empty results with total=0. Don't raise an error.
    - Let SQLAlchemy exceptions propagate naturally.

    **Performance notes:**
    - The GIN index on `question_tsv` (from Plan 01 migration) ensures sub-200ms on 1000+ rows.
    - `plainto_tsquery` handles natural language input safely (no injection risk from raw tsquery syntax).
    - The count query may be slower on very large datasets -- acceptable for v2.1 scale.
