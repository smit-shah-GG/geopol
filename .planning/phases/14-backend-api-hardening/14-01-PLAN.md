       ```python

       ```

    b) Modify `get_top_forecasts` (line ~121-159):
       - Remove the bare `except Exception` that swallows PostgreSQL errors. Replace with proper error handling: let the exception propagate (FastAPI's error handler will return 500).
       - Gate the fixture fallback:
         ```python
         # PostgreSQL query
         service = ForecastService(db)
         result = await service.get_top_forecasts(limit=limit)
         if result:
             data = [item.model_dump(mode="json") for item in result]
             await cache.set(key, data, ttl=SUMMARY_TTL)
             return result

         # Fixture fallback (dev only)
         settings = get_settings()
         if settings.use_fixtures:
             fixture_cache = _get_fixture_cache()
             forecasts = sorted(fixture_cache.values(), key=lambda f: f.probability, reverse=True)
             return forecasts[:limit]

         return []  # No data = empty list
         ```

    c) Modify `get_forecasts_by_country` (line ~162-236):
       - Same pattern: remove bare except, gate fixture fallback behind `use_fixtures`, return empty PaginatedResponse when no data.

       - When `use_fixtures=False` and PostgreSQL returns empty:
         ```python
         return PaginatedResponse[ForecastResponse](items=[], next_cursor=None, has_more=False)
         ```

    d) Modify `get_forecast_by_id` (line ~239-281):
       - Same pattern: remove bare except, gate fixture fallback, raise 404 when not found.
       - When `use_fixtures=False`: cache -> PostgreSQL -> 404. No fixture step.

