  <action>
    Create src/api/services/submission_worker.py implementing the background forecast processing worker.

    **Architecture:** asyncio.create_task + Semaphore(3) for bounded parallelism. No external dependencies (no Celery, no Redis queue). The worker runs in the same event loop as FastAPI.

    **Module-level state:**
    ```python
    import asyncio
    import logging
    from datetime import datetime, timezone

    from sqlalchemy import select, update
    from sqlalchemy.ext.asyncio import AsyncSession

    from src.db.models import ForecastRequest
    from src.db.postgres import get_async_session

    logger = logging.getLogger(__name__)

    _worker_semaphore = asyncio.Semaphore(3)
    _active_tasks: set[asyncio.Task] = set()

    MAX_RETRIES = 3
    RETRY_DELAYS = [30, 120, 600]  # seconds: 30s, 2min, 10min
    ```

    **`schedule_processing(request_id: str)` function:**
    - Creates an asyncio task for `_process_request(request_id)`
    - Adds to `_active_tasks` set (prevents GC)
    - Task removes itself from set on completion via `add_done_callback`

    **`_process_request(request_id: str)` coroutine:**
    ```python
    async def _process_request(request_id: str) -> None:
        async with _worker_semaphore:
            # Get a fresh DB session (not the request's session -- it's closed)
            async for session in get_async_session():
                try:
                    await _execute_forecast(session, request_id)
                except Exception as exc:
                    logger.error("Worker failed for request %s: %s", request_id, exc, exc_info=True)
                    await _handle_failure(session, request_id, str(exc))
    ```

    **`_execute_forecast(session, request_id)` coroutine:**

    1. Load ForecastRequest from DB. Verify status is "confirmed" or "processing" (idempotency).
    2. Update status to "processing".
    3. Check Gemini budget via `gemini_budget_remaining()`:
       ```python
       from src.api.middleware.rate_limit import gemini_budget_remaining
       from src.api.deps import get_redis
       redis_client = await get_redis()
       remaining = await gemini_budget_remaining(redis_client)
       if remaining <= 0:
           raise BudgetExhaustedError("Gemini daily budget exhausted")
       ```
    4. For each country_iso in request.country_iso_list:
       a. Import EnsemblePredictor lazily (same pattern as POST /forecasts route)
       b. Run `await asyncio.to_thread(predictor.predict, question=request.question)`
       c. Persist via ForecastService.persist_forecast() with the country_iso
       d. Append prediction.id to request.prediction_ids
       e. Increment Gemini usage counter
    5. Update status to "complete", set completed_at.
    6. Commit.

    **`_handle_failure(session, request_id, error_msg)` coroutine:**

    1. Load request, increment retry_count.
    2. If retry_count < MAX_RETRIES:
       - Set status back to "confirmed" (worker will re-pick)
       - Set error_message with retry info
       - Schedule retry with delay: `asyncio.get_event_loop().call_later(RETRY_DELAYS[retry_count], lambda: asyncio.create_task(_process_request(request_id)))`
    3. If retry_count >= MAX_RETRIES:
       - Set status to "failed"
       - Set error_message to final error
    4. Commit.

    **Custom exception:**
    ```python
    class BudgetExhaustedError(Exception):
        """Raised when Gemini API daily budget is exhausted."""
        pass
    ```

    **Race condition prevention:**
    The confirm endpoint sets status to "confirmed" and calls schedule_processing. The worker atomically checks status == "confirmed" before setting to "processing". Use `SELECT ... FOR UPDATE` when loading the request in _execute_forecast to prevent double-pickup:

    ```python
    stmt = (
        select(ForecastRequest)
        .where(ForecastRequest.id == request_id)
        .with_for_update(skip_locked=True)
    )
    result = await session.execute(stmt)
    request = result.scalar_one_or_none()
    if request is None or request.status not in ("confirmed", "processing"):
        logger.info("Request %s already claimed or not ready, skipping", request_id)
        return
    ```

    **Important implementation details:**
    - The worker gets its OWN database session via `get_async_session()` -- NOT the request handler's session. The HTTP request's session is closed by the time the worker runs.
    - EnsemblePredictor is imported lazily inside the worker to avoid circular imports.
    - The predictor.predict() call is synchronous and CPU-bound -- wrap in `asyncio.to_thread()`.
    - Each country in country_iso_list gets a separate EnsemblePredictor.predict() call.
    - Log extensively: start, budget check, per-country progress, completion, failures.
