  <action>
    Major rewrite of `renderTree()` in ScenarioExplorer.ts. The current implementation (lines 242-378) uses fixed 200x100px node spacing, SVG `<text>` with `truncate(40)`, and no pan/zoom. Replace with:

    **1. foreignObject multi-line text blocks:**
    Replace the SVG `<text>` description label (lines 336-343) with `<foreignObject>` containing an HTML `<div>`:
    - Width: ~160px. Height: ~60px (enough for 2-3 lines at 12-13px font).
    - Text content: `node.data.name.slice(0, 120)` -- show up to 120 chars with CSS word-wrap.
    - CSS class: `scenario-node-text` with styles: `font-size: 12px; line-height: 1.3; color: var(--text-secondary); word-wrap: break-word; overflow: hidden;`
    - Keep the probability label as SVG `<text>` above the circle (unchanged).
    - Keep tooltip on hover for full description when text exceeds 120 chars.

    **2. Alternating sides layout:**
    After `treeLayout(root)` computes node positions, post-process to determine subtree side:
    - For each child of root, determine if it's in the "left half" or "right half" based on its x position relative to root.x.
    - Left-subtree nodes: text block positioned to the LEFT of the node circle (`x = -(textWidth + radius + 8)`).
    - Right-subtree nodes: text block positioned to the RIGHT (`x = radius + 8`).
    - Root node: text centered below (or skip text block for root -- root content goes to sidebar).
    - Store the side determination on the TreeDatum interface: add `side?: 'left' | 'right'` field.

    **3. Dynamic node separation:**
    Replace `d3.tree<TreeDatum>().nodeSize([200, 100])` with dynamic spacing:
    ```typescript
    const treeLayout = d3.tree<TreeDatum>()
      .nodeSize([250, 140])
      .separation((a, b) => {
        // Wider separation for nodes with long text
        return a.parent === b.parent ? 1.2 : 1.5;
      });
    ```
    Increase vertical spacing from 100 to 140 to accommodate multi-line text. Increase horizontal from 200 to 250.

    **4. Bezier curves (already present):**
    `d3.linkVertical()` already produces cubic beziers in d3 v7. The current code uses it correctly. Optionally increase stroke-width to 2 and opacity to 0.7 for better visibility with the wider layout. No functional change needed -- just cosmetic enhancement.

    **5. Pan/zoom via d3.zoom():**
    - Wrap all tree content (links + nodes) in a `<g>` group element instead of appending directly to SVG.
    - Apply `d3.zoom()` to the SVG element, transforming the inner group:
      ```typescript
      import { zoom as d3zoom, select } from 'd3';

      const svgGroup = document.createElementNS('http://www.w3.org/2000/svg', 'g');
      // ... append all links and nodes to svgGroup ...
      svg.appendChild(svgGroup);

      const zoomBehavior = d3zoom<SVGSVGElement, unknown>()
        .scaleExtent([0.3, 3])
        .filter((event: Event) => {
          // Only enable zoom when tree has 5+ scenario nodes (per context decision)
          // Also filter out wheel events when modal body needs scrolling
          if (event.type === 'wheel') {
            return nodes.length >= 5;
          }
          return true;
        })
        .on('zoom', (event) => {
          svgGroup.setAttribute('transform', event.transform.toString());
        });

      select(svg).call(zoomBehavior as any);
      ```
    - The `.filter()` prevents d3-zoom from capturing wheel events on small trees (per Pitfall 6 in RESEARCH.md), allowing modal scroll to work normally.

    **6. CSS additions in panels.css:**
    Add styles for the new text blocks:
    ```css
    .scenario-node-text {
      font-size: 12px;
      line-height: 1.3;
      color: var(--text-secondary);
      word-wrap: break-word;
      overflow: hidden;
      pointer-events: none;
    }
    .scenario-node-text.left-side {
      text-align: right;
    }
    .scenario-node-text.right-side {
      text-align: left;
    }
    ```

    IMPORTANT: Do NOT break the existing click-to-select-node behavior. The `g.addEventListener('click', ...)` that calls `selectNode()` must remain on the node group element, not on the foreignObject.
  </action>
