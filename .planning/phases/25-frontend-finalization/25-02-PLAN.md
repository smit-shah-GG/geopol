Apply a consistent three-tier pattern to every Panel subclass. The pattern for each panel's `refresh()` method (or equivalent fetch method) is:

**Initial load (no stale data):**
```
async refresh(): Promise<void> {
  // Don't show skeleton if we already have content (refresh cycle)
  if (!this.hasData) {
    this.showSkeleton();
  }
  try {
    const data = await forecastClient.getXxx();
    this.hasData = true;
    this.dismissToast();
    this.renderXxx(data);
  } catch (err) {
    if (this.isAbortError(err)) return;
    if (this.hasData) {
      // Stale data visible -- show toast, keep content
      const severity = isTransient(err) ? 'amber' : 'red';
      this.showRefreshToast('Failed to refresh -- showing cached data', severity);
    } else {
      // No data at all -- show error with retry
      this.showErrorWithRetry('Unable to load data', () => this.refresh());
    }
  }
}
```

Add a `private hasData = false;` field to each panel to track whether real content has been rendered at least once.

Helper for error severity: if the error message includes "timeout", "503", "502", "504", "ECONNREFUSED", or "network", treat as transient (amber). Otherwise persistent (red). This can be a local function or inline check.
