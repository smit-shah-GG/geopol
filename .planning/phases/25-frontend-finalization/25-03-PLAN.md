**Focus trapping in ScenarioExplorer:**

Import `trapFocus` from `@/utils/focus-trap`. Read the file to find the `open()` method and the modal container element.

In the `open()` method, after the modal becomes visible (after DOM is populated):
1. Call `this.releaseTrap = trapFocus(this.modal)` (store as `private releaseTrap: (() => void) | null = null`)
2. In the `close()` method, call `this.releaseTrap?.()` and set to null

Add `role="dialog"` and `aria-modal="true"` and `aria-label="Scenario Explorer"` to the modal container element.
Add `aria-label="Close scenario explorer"` to the close button.

**Focus trapping in CountryBriefPage:**

Same pattern as ScenarioExplorer. Read the file to find the modal/overlay element and open/close methods.

1. Import `trapFocus` from `@/utils/focus-trap`
2. In `open()`: `this.releaseTrap = trapFocus(this.overlay)`
3. In `close()`: `this.releaseTrap?.(); this.releaseTrap = null;`
4. Add `role="dialog"`, `aria-modal="true"`, `aria-label="Country Brief"` to overlay
5. Add `aria-label="Close country brief"` to close button
6. Tab bar buttons: add `role="tab"`, `aria-selected` toggled on active tab

**Focus trapping in SettingsModal:**

Same pattern. Read the file to find the modal element and open/close methods.

1. Import `trapFocus` from `@/utils/focus-trap`
2. In the method that shows the modal: `this.releaseTrap = trapFocus(this.modal)`
3. In close: `this.releaseTrap?.(); this.releaseTrap = null;`
4. Add `role="dialog"`, `aria-modal="true"`, `aria-label="Settings"` to modal
5. Add `aria-label="Close settings"` to close button

**LayerPillBar ARIA labels:**

Read the file. Each pill button needs `aria-label` and `aria-pressed` attributes:

For each pill in `LAYER_PILLS`:
- Add `aria-label: "Toggle {label} layer"` (e.g., "Toggle Risk layer", "Toggle Arcs layer")
- Add `aria-pressed: String(this.states[id])` -- "true" when active, "false" when not
- In the `toggle()` method, update `aria-pressed` on the pill element after toggling

Also add `role="toolbar"` and `aria-label="Map layers"` to the pill bar container.

**NavBar ARIA:**

Read the file. The nav element likely already has semantic `<nav>`, but add:
- `aria-label="Main navigation"` on the `<nav>` element
- Each link should have `aria-current="page"` when active (in the `updateActive` function, toggle this attribute)

**GlobeHud ARIA:**

Read the file. Add `aria-label` to any interactive elements (if any exist). The HUD is likely display-only, so this may be minimal. Add `role="status"` and `aria-live="polite"` on the HUD container so screen readers announce updates.

**DeckGLMap ARIA + resize fix:**

Read the file. Find the map container and any zoom/control buttons.

1. Add `aria-label="Geopolitical forecast map"` and `role="application"` to the map container
2. If zoom buttons exist (MapLibre NavigationControl), they should already have ARIA labels from MapLibre. If custom zoom buttons exist, add `aria-label="Zoom in"` and `aria-label="Zoom out"`
3. **Globe resize fix:** In the method where the map is initialized, add a `requestAnimationFrame` or `setTimeout(0)` after map construction that calls `this.map.resize()`. Also add a `ResizeObserver` on the container that calls `this.map.resize()` when dimensions change. This fixes the initial-mount sizing issue.

**globe-screen.ts resize trigger:**

After the DeckGLMap is mounted into the container, ensure `map.resize()` is called. Read the mountGlobe function to find where deckMap is constructed and mounted. Add a `requestAnimationFrame(() => deckMap?.resize())` after the container is appended to the DOM (if DeckGLMap exposes a resize method; if not, the ResizeObserver in DeckGLMap itself handles it).

**Keyboard navigation for expand/collapse:**

Read `expandable-card.ts` to find the card toggle handler. The card header (or the clickable element) needs:
- `role="button"` and `tabindex="0"` on the card header/toggle element
- `aria-expanded="true|false"` toggled on expand/collapse
- `keydown` listener on Enter and Space that triggers the same toggle as click
