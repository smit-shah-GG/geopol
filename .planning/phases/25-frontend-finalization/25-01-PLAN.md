**skeleton.ts** -- `buildSkeleton(shape: SkeletonShape): HTMLElement`

Export a `SkeletonShape` type union: `'card-list' | 'row-list' | 'bar-pairs' | 'text-block' | 'news-feed' | 'health-grid' | 'timeline'`.

The function creates a container `div.panel-skeleton` with child elements matching each shape:
- `card-list`: 3 stacked `.skeleton-card` blocks (each ~80px tall with a `.skeleton-line` title bar + `.skeleton-bar` probability bar)
- `row-list`: 6 narrow `.skeleton-row` elements (each ~32px, simulating RiskIndex rows with flag placeholder + score + text)
- `bar-pairs`: 3 pairs of `.skeleton-bar-pair` (two horizontal bars per pair, simulating ComparisonPanel dual bars)
- `text-block`: 4 `.skeleton-line` elements of varying widths (80%, 100%, 60%, 90%) for SystemHealth/generic text
- `news-feed`: 5 `.skeleton-article` blocks (each with `.skeleton-line` headline + 2x `.skeleton-line.short` sublines)
- `timeline`: 4 `.skeleton-timeline-item` blocks (dot + 2 lines, for EventTimeline)
- `health-grid`: 1 `.skeleton-line.wide` + 4 `.skeleton-row` narrow items (for SystemHealth)

Use `h()` from `@/utils/dom-utils` for all DOM construction. No innerHTML.

Also export a `PANEL_SKELETON_MAP: Record<string, SkeletonShape>` mapping panel IDs to their shapes:
- `forecasts` -> `card-list`
- `risk-index` -> `row-list`
- `news-feed` -> `news-feed`
- `system-health` -> `health-grid`
- `polymarket` -> `card-list`
- `my-forecasts` -> `card-list`
- `comparisons` -> `bar-pairs`
- `event-timeline` -> `timeline`
- `live-streams` -> `text-block`
- `sources` -> `row-list`

**focus-trap.ts** -- Reusable modal focus trapping.

Export `trapFocus(container: HTMLElement): () => void` that:
1. Stores the currently focused element as `previousFocus`
2. Queries all focusable elements inside `container` (inputs, buttons, anchors, [tabindex], selects, textareas) excluding `[tabindex="-1"]` and `[disabled]`
3. Adds a `keydown` listener on `container` that:
   - On Tab: cycles focus to first focusable if at last, or to last if Shift+Tab at first
   - Re-queries focusable elements on each Tab press (handles dynamic content)
4. Focuses the first focusable element (or the container itself if none found, setting tabindex=-1)
5. Returns a `releaseFocus` cleanup function that removes the keydown listener and restores focus to `previousFocus`

Export `releaseFocus` as a no-op for import convenience (the real release is the returned function).

**timing.ts** -- Extract debounce/throttle from SearchBar into shared utility.

Export `debounce<T extends (...args: any[]) => void>(fn: T, ms: number): T` -- standard trailing-edge debounce with clearTimeout/setTimeout pattern. Copy from SearchBar.ts lines 20-33.

Export `throttle<T extends (...args: any[]) => void>(fn: T, ms: number): T` -- standard leading-edge throttle that fires immediately then suppresses for `ms`.

**Panel.ts changes:**

Add imports: `import { buildSkeleton, PANEL_SKELETON_MAP } from '@/utils/skeleton';`

Add three new public methods to the Panel class:

1. `showSkeleton(): void` -- Replaces content with skeleton matching this panel's ID. Looks up `PANEL_SKELETON_MAP[this.panelId]`, falls back to `'text-block'` if not found. Calls `replaceChildren(this.content, buildSkeleton(shape))`. This replaces the radar sweep animation for initial loads.

2. `showErrorWithRetry(message: string, retryFn: () => void | Promise<void>): void` -- Replaces content with a centered error block: `.panel-error-block` containing an error icon (unicode cross mark or exclamation), the message text in `.panel-error-message`, and a `button.panel-retry-btn` labeled "Retry". The button's click handler calls `retryFn()`. This is for initial load failures (no stale data available).

3. `showRefreshToast(message: string, severity: 'amber' | 'red' = 'amber'): void` -- Inserts (or replaces) a `.panel-refresh-toast` element at the TOP of `this.content` (using `insertBefore` before first child), without clearing existing content. The toast has class `toast-${severity}` for color. Adds a dismiss button (x) that removes the toast. Sets a 10-second auto-dismiss timer via setTimeout. If a toast already exists in content, remove it before inserting the new one (prevent accumulation).

4. `dismissToast(): void` -- Removes any existing `.panel-refresh-toast` from content. Called by subclasses after successful refresh.

Modify the constructor: Change `this.showLoading()` at line 137 to `this.showSkeleton()`. This makes all panels start with a skeleton instead of the radar sweep.

Keep the existing `showLoading()`, `showError()`, and `showRetrying()` methods intact for backward compatibility -- subclasses that haven't been upgraded yet still call them.
