  <name>Task 1: Cross-source dedup module + graph builder integration</name>
  <files>
    src/knowledge_graph/cross_source_dedup.py
    src/knowledge_graph/graph_builder.py
  </files>
  <action>
    1. Create `src/knowledge_graph/cross_source_dedup.py`:
       - `cross_source_fingerprint(event_date: str, country_iso: str | None, event_type: str) -> str`: SHA-256 hash of `"{date}|{country}|{coarse_type}"`, truncated to 32 hex chars. Normalize: date to YYYY-MM-DD, country to uppercase or "UNK", event_type to uppercase.
       - `cameo_to_coarse_type(cameo_code: str) -> str`: Map CAMEO 2-digit prefix to coarse categories. Prefix 01-05 -> "cooperation", 06-09 -> "diplomacy", 10-14 -> "conflict", 15-20 -> "force". Handle empty/invalid codes -> "unknown".
       - `CrossSourceDedupFilter`: Class that maintains an in-memory dict of `{fingerprint: (source, event_id)}` for the current graph-building session. Method `should_insert(event_date, country_iso, cameo_code, source: str, event_id: str) -> bool`:
         - Compute fingerprint.
         - If fingerprint not seen: record it, return True.
         - If fingerprint seen from SAME source: return True (intra-source dedup handled elsewhere).
         - If fingerprint seen from DIFFERENT source: prefer ACLED (if either is ACLED, keep ACLED's). Log suppressed duplicate: `logger.info("Cross-source dedup: suppressing %s event %s (duplicate of %s event %s, fingerprint %s)", ...)`. Return True/False depending on which source wins.
       - Add `stats` property: dict with counts of {checked, suppressed, by_source_pair}.

    2. Integrate into `src/knowledge_graph/graph_builder.py`:
       - In `TemporalKnowledgeGraph.__init__()`, create a `CrossSourceDedupFilter` instance.
       - In `add_event_from_db_row()` (or whatever method inserts events), call `self.dedup_filter.should_insert(...)` before creating the graph triple. Skip insertion if returns False.
       - After a batch build completes, log dedup stats: `logger.info("Cross-source dedup stats: %s", self.dedup_filter.stats)`.

    IMPORTANT: This is CONSERVATIVE dedup. The (date, country, event_type) fingerprint WILL miss some duplicates (same country, same day, different coarse type). That is intentional — false merges are worse than missed dedup. The audit log enables future tuning.
  </action>
  <verify>

Unit test the dedup module: `python -c "from src.knowledge_graph.cross_source_dedup import cross_source_fingerprint, cameo_to_coarse_type; print(cameo_to_coarse_type('14')); print(cross_source_fingerprint('2026-01-01', 'US', 'conflict'))"` should print "conflict" and a 32-char hex string.
  </verify>
  <done>CrossSourceDedupFilter integrated into graph_builder. Fingerprint-based dedup at graph insertion with ACLED preference, full audit logging, and session stats.</done>
