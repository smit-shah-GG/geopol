  <action>
    1. **Add types to admin-types.ts**:
       - `FeedInfo`: id, name, url, tier, category, lang, enabled, last_poll_at (string|null), last_error (string|null), error_count, articles_24h, articles_total, avg_articles_per_poll, created_at (string).
       - `AddFeedRequest`: name, url, tier (1|2), category?, lang?.
       - `UpdateFeedRequest`: Partial of name, url, tier, category, lang, enabled.

    2. **Add AdminClient methods** to `admin-client.ts`:
       - `getFeeds(): Promise<FeedInfo[]>` — GET /api/v1/admin/feeds
       - `addFeed(data: AddFeedRequest): Promise<FeedInfo>` — POST /api/v1/admin/feeds
       - `updateFeed(id: number, data: UpdateFeedRequest): Promise<FeedInfo>` — PUT /api/v1/admin/feeds/{id}
       - `deleteFeed(id: number, purge?: boolean): Promise<void>` — DELETE /api/v1/admin/feeds/{id}?purge={purge}
       All use the existing `this.fetch()` pattern with X-Admin-Key header.

    3. **Rewrite SourceManager** (`frontend/src/admin/panels/SourceManager.ts`):
       Replace the current simple card grid (which reads from `/api/v1/admin/sources`) with a rich feed management panel that reads from `/api/v1/admin/feeds`:

       - **Header bar**: "Feed Management" title + "Add Feed" button.
       - **Add Feed form** (expandable, hidden by default): URL input, name input, tier dropdown (1/2), category dropdown, submit button. Calls `client.addFeed()`. On success, refresh the grid and show toast.
       - **Feed card grid** (2-3 columns): Each card shows:
         - Feed name (bold) + tier badge (T1 gold, T2 blue).
         - URL (truncated, tooltip on hover).
         - Health dot (green if error_count=0, yellow if 1-4, red if >=5 or auto-disabled).
         - Last poll: relative time from last_poll_at.
         - Articles: "X today / Y total" from articles_24h and articles_total.
         - Avg/poll: avg_articles_per_poll rounded to 1 decimal.
         - Last error: if last_error is non-null, show in red text (truncated to 80 chars, full on hover).
         - Error count: show if > 0, with warning color.
         - Enable/disable toggle (existing pattern, now calls `client.updateFeed(id, { enabled })`).
         - Tier selector: dropdown or toggle to change tier (calls `client.updateFeed(id, { tier })`).
         - Delete button: confirmation prompt, then calls `client.deleteFeed(id)`. Offer "Delete + Purge Articles" option.
       - **Auto-disabled alert**: If any feed has `enabled=false` AND `error_count >= 5`, show a yellow banner at top: "N feeds auto-disabled due to repeated errors. Review below."
       - **Stats footer**: Total feeds, enabled feeds, total articles today, feeds with errors.
       - Auto-refresh every 30s (existing interval pattern).

    4. **Admin CSS** (`admin-styles.css`): Add styles for feed cards, add-feed form, tier selector, error text, auto-disabled banner, stats footer. Keep consistent with existing admin panel dark theme.

    NOTE: The old SourceManager read from `client.getSources()` (which hits `/api/v1/admin/sources` returning SourceInfo). The new one reads from `client.getFeeds()` (which hits `/api/v1/admin/feeds` returning FeedInfo). The old `getSources()` and `toggleSource()` methods can remain for backward compatibility but are no longer the primary data source for this panel.

       - Clicking the label shows a tooltip or small dropdown listing which sources are unhealthy and their detail message.

    IMPORTANT: The source health indicator uses the PUBLIC `/api/v1/sources` endpoint (no admin key needed). It shows aggregated daemon health (gdelt, rss, acled, advisory), not per-feed granularity. Per-feed health is admin-only.
