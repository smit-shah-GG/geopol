**Layer 1 -- Forecast Risk Choropleth (updateRiskScores):**
- Use `GeoJsonDataSource.load()` with the GeoJSON from `countryGeometry.getGeoJSON()`.
- Default styling: `stroke: Color.fromCssColorString('rgba(80,85,95,0.6)')`, `strokeWidth: 1`, `fill: Color.fromCssColorString('rgba(40,44,52,0.47)')`, `clampToGround: true`.
- After load, iterate entities: extract ISO code from properties (same normalization as existing code in DeckGLMap.ts -- check `properties.ISO_A2` or `properties.ADM0_A3`), store in `entityIsoMap`, tag with `entity.properties.addProperty('layerId', 'ForecastRiskChoropleth')` and `entity.properties.addProperty('iso', iso)`.
- Color by risk score: blue-to-red diverging scale. `riskColor(score)` helper that returns a CesiumJS `Color`:
  - score 0-0.3: blue tones `rgba(40,80,180,0.5)`
  - score 0.3-0.6: yellow tones `rgba(200,180,40,0.5)`
  - score 0.6-1.0: red tones `rgba(220,50,50,0.5)`
  Use a smooth interpolation, not hard cutoffs. Match the existing DeckGLMap color ramp.
- On re-call of `updateRiskScores()`, iterate `entityIsoMap` and update materials. Do NOT reload GeoJSON -- just re-color existing entities. First call loads GeoJSON; subsequent calls only re-color.
- Set `entity.show` based on `layerVisibility['ForecastRiskChoropleth']`.
- Highlight selected country: bright outline. Use a `polyline` entity or change `polygon.outlineColor` + `polygon.outlineWidth` on the selected entity.
- Call `viewer.scene.requestRender()` after update.

**Layer 2 -- Active Forecast Markers (updateForecasts):**
- Clear existing billboards: `billboards.removeAll()`.
- For each forecast, get centroid from `countryGeometry.getCentroid()`. Skip if no centroid.
- Create canvas-rendered marker sprite (pin icon): 32x32 canvas, draw a pin/circle shape color-coded by probability. Higher probability = more red, lower = more blue/green.
- Add billboard: `billboards.add({ position: Cartesian3.fromDegrees(lng, lat), image: canvas, id: { layerId: 'ActiveForecastMarkers', iso, data: forecast }, scale: 1.0, show: layerVisibility['ActiveForecastMarkers'] })`.
- Call `viewer.scene.requestRender()`.

**Layer 3 -- Knowledge Graph Arcs (updateArcData):**
- Clear existing polylines: `polylines.removeAll()`.
- For each arc, generate parabolic positions using `generateArcPositions()` helper (RESEARCH.md Pattern 2):
  - 40 segments, `4t(1-t)` parabola
  - Peak height: `Math.max(100_000, Math.min(2_000_000, surfaceDistDeg * 50_000))`
  - Use `Cartesian3.fromDegreesArrayHeights()` for position array
- Color by avgGoldstein: negative = `rgba(255,80,80,0.7)` (conflictual), positive = `rgba(80,180,255,0.7)` (cooperative).
- Width: `Math.max(1, Math.min(5, arc.eventCount / 20))`.
- `id: { layerId: 'KnowledgeGraphArcs', data: arc }`.
- Set `show` based on `layerVisibility['KnowledgeGraphArcs']`.
- Call `viewer.scene.requestRender()`.

**Layer 4 -- GDELT Event Heatmap (updateHeatmapData):**
- Lazy-load h3-js: `if (!this.h3Module) this.h3Module = await import('h3-js');` -- make `updateHeatmapData` async or use a cached promise pattern (same as GlobeMap.ts approach).
- Clear existing points: `points.removeAll()`.
- Find `maxWeight` for normalization.
- For each hexbin: `const [lat, lng] = this.h3Module.cellToLatLng(hex.h3_index)`. NaN guard: skip if `!Number.isFinite(lat) || !Number.isFinite(lng)`.
- Add point: `points.add({ position: Cartesian3.fromDegrees(lng, lat, 0), color: heatColor(t), pixelSize: 6 + t * 6, id: { layerId: 'GDELTEventHeatmap', data: hex }, show: layerVisibility['GDELTEventHeatmap'] })`.
- Heat color: cool-to-hot gradient. `t = min(1, weight / maxWeight)`. Color: `rgba(255, ${round(255*(1-t))}, 0, ${0.4 + 0.5*t})`.
- Call `viewer.scene.requestRender()`.

- If using a separate GeoJsonDataSource for scenarios: load same GeoJSON, but only show entities whose ISO has delta data. Set `polygon.height = 200`, color by delta.
- ALTERNATIVE (simpler, recommended): For scenario zones, create standalone Entity instances added to `viewer.entities` directly. For each country with delta data, get the GeoJSON polygon coordinates from `countryGeometry.getFeature(iso)`, create an Entity with `polygon: { hierarchy: ..., height: 200, material: deltaColor }`. Tag with `{ layerId: 'ScenarioZones', iso, delta }`.
- Actually the cleanest approach: store scenario zone entities in a separate array, remove/recreate on data update. Or use a dedicated `CustomDataSource` for scenario zones.
- Set show based on `layerVisibility['ScenarioZones']`.

**Scene mode methods:**
```typescript
setSceneMode(mode: '3d' | 'columbus' | '2d'): void {
  switch (mode) {
    case '3d': this.viewer.scene.morphTo3D(2.0); break;
    case 'columbus': this.viewer.scene.morphToColumbusView(2.0); break;
    case '2d': this.viewer.scene.morphTo2D(2.0); break;
  }
  localStorage.setItem('geopol-globe-mode', mode);
}

getSceneMode(): '3d' | 'columbus' | '2d' {
  switch (this.viewer.scene.mode) {
    case SceneMode.SCENE3D: return '3d';
    case SceneMode.COLUMBUS_VIEW: return 'columbus';
    case SceneMode.SCENE2D: return '2d';
    default: return '3d';
  }
}
```
