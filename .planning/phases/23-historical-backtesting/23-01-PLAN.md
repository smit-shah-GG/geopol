  <action>
    **src/backtesting/evaluator.py:**
    Metric computation functions (stateless, pure numpy):
    - `compute_brier_score(predictions: list[float], outcomes: list[float]) -> float` -- `np.mean((np.array(predictions) - np.array(outcomes)) ** 2)`
    - `compute_calibration_bins(predictions: list[float], outcomes: list[float], n_bins: int = 10) -> dict` -- Returns {"bins": [...], "predicted_avg": [...], "observed_freq": [...], "counts": [...]}. Use np.histogram for bucketing. Handle empty bins gracefully (None for avg/freq when count is 0).
    - `compute_hit_rate(predictions: list[float], outcomes: list[float], threshold: float = 0.5) -> dict` -- Returns {"total": N, "correct": M, "hit_rate": M/N}. A prediction is "correct" if (predicted >= threshold and outcome == 1.0) or (predicted < threshold and outcome == 0.0).
    - `compute_mrr(rankings: list[int]) -> float` -- Mean reciprocal rank from TKG predictions. If rankings list is empty, return 0.0.

    **src/backtesting/temporal_index.py:**
    Ephemeral ChromaDB index builder (prevents look-ahead bias -- BTEST-05):
    - `build_temporal_chromadb_index(source_persist_dir: str, source_collection_name: str, cutoff_date: datetime) -> chromadb.Collection`
    - Opens the persistent ChromaDB source (read-only access), creates an ephemeral in-memory client via `chromadb.Client()`, copies all chunks with published_at <= cutoff_date using paginated get() with BATCH_SIZE=500.
    - CRITICAL: Filter out chunks with empty/missing published_at metadata (exclude to be safe -- no date means we can't verify temporal validity).
    - CRITICAL: Normalize published_at comparison in Python using `datetime.fromisoformat()` rather than relying solely on ChromaDB's string $lte (date format inconsistency pitfall from RESEARCH.md).
    - Use `SentenceTransformerEmbeddingFunction(model_name="sentence-transformers/all-mpnet-base-v2")` for both source and temp collection (must match).

    **src/backtesting/weight_snapshot.py:**
    Calibration weight time-travel (prevents look-ahead bias -- BTEST-05):
    - `async snapshot_calibration_weights(session: AsyncSession, as_of: datetime) -> dict[str, float]`
    - Queries `calibration_weight_history` for the latest `auto_applied=True` entry per `cameo_code` where `computed_at <= as_of`.
    - Falls back to cold-start priors from `src/calibration/priors.py` for codes with no history before `as_of`. Import the priors module to get default alpha values.
    - Returns dict of cameo_code -> alpha suitable for injection into WeightLoader's _weights cache.

    **src/backtesting/runner.py:**
    Central BacktestRunner class:
    - `__init__(self, run_config: BacktestRunConfig, async_session_factory)` -- stores config, session factory.
    - `async run(self) -> BacktestRunResult` -- Main orchestration loop:
      1. Query the predictions table for the earliest and latest prediction dates to determine the overall date range.
      2. Generate sliding windows: window = [t0, t0 + window_size], prediction_window = [t0 + window_size, t0 + 2*window_size], slide by slide_step. Each window pair = (train_end, predict_start, predict_end).
      3. Update backtest_runs row: status='running', started_at=now, total_windows=len(windows).
      4. For each window (and for each checkpoint in config.checkpoints):
         a. Check cancellation: query `backtest_runs.status` from DB; if 'cancelling', set status='cancelled', return partial results.
         b. Snapshot calibration weights at window end via `snapshot_calibration_weights()`.
         c. Build ephemeral ChromaDB temporal index via `build_temporal_chromadb_index()` with cutoff_date = window_end.
         d. Query predictions from the prediction window [prediction_start, prediction_end] that have resolved outcomes (join with outcome_records).
         e. If fewer than min_predictions_per_window resolved predictions, skip this window (log it).
         f. For each prediction: build a fresh EnsemblePredictor (following _build_predictor pattern from auto_forecaster.py), inject the weight snapshot and temporal ChromaDB index, re-predict. Collect predicted probability and actual outcome.
         g. Compute metrics via evaluator functions (Brier, calibration bins, hit rate).
         h. If prediction has polymarket_comparison data, also compute polymarket Brier and head-to-head.
         i. Persist WindowResult as BacktestResult row.
         j. Update backtest_runs.completed_windows.
      5. After all windows: compute aggregate metrics (mean Brier, mean MRR, total vs_polymarket record).
      6. Update backtest_runs: status='completed', completed_at=now, aggregate_brier, aggregate_mrr, vs_polymarket_record_json.

    IMPORTANT patterns to follow:
    - Fresh EnsemblePredictor per prediction call (mutable _forecast_output state prevents reuse).
    - DB-based cancellation: poll backtest_runs.status between windows, NOT threading.Event (process-safe).
    - Cache heavy components (RAG pipeline, TKG predictor) across predictions within a window, but create fresh EnsemblePredictor each time.
    - The runner creates its own DB engine/session via `_pg.init_db()` when called from ProcessPoolExecutor (no pickling of sessions).
    - Handle the case where `_build_predictor()` needs a checkpoint_path override for RE-GCN vs TiRGN comparison (BTEST-02 support).

    **src/backtesting/export.py:**
    Export functions:
    - `async export_run_json(session: AsyncSession, run_id: str) -> dict` -- Fetches BacktestRun + all BacktestResults, returns a dict with run metadata, all window results, aggregate metrics, and a "methodology" key containing the methodology template.
    - `async export_run_csv(session: AsyncSession, run_id: str) -> str` -- Returns CSV string with headers: window_start, window_end, checkpoint, num_predictions, brier_score, mrr, hits_at_1, hits_at_10, polymarket_brier, geopol_wins, pm_wins. Include methodology as a comment block at the top.
    - `async export_multi_run_json(session: AsyncSession, run_ids: list[str]) -> dict` -- Merges multiple runs into a single export with run labels as group identifiers.
    - `METHODOLOGY_TEMPLATE: str` -- Constant string documenting: (1) Walk-forward evaluation methodology (window size, slide step, overlap), (2) Bias prevention measures (temporal ChromaDB indexes, calibration weight snapshots), (3) Metric definitions (Brier score formula, MRR, Hits@k, reliability diagram bins), (4) Inclusion criteria (minimum predictions per window), (5) Data sources (all predictions regardless of provenance -- organic + polymarket-driven).
