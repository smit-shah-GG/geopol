    **1a. Add search/submission types to api.ts:**
    Add the following TypeScript interfaces (match backend schemas exactly):
    ```typescript
    /** GET /forecasts/search result item */
    export interface SearchResult {
      forecast: ForecastResponse;
      relevance: number;
    }

    /** GET /forecasts/search response */
    export interface SearchResponse {
      results: SearchResult[];
      total: number;
      query: string;
      suggestions: string[] | null;
    }
    ```
    These types are used by the search endpoint (BAPI-04, built in Phase 14).

    **1b. Add search() method to forecast-client.ts:**
    Add a `search()` method to `ForecastServiceClient`:
    ```typescript
    async search(
      q: string,
      options?: { country?: string; category?: string; limit?: number; signal?: AbortSignal },
    ): Promise<SearchResponse> {
      const params = new URLSearchParams({ q });
      if (options?.country) params.set('country', options.country);
      if (options?.category) params.set('category', options.category);
      if (options?.limit) params.set('limit', String(options.limit));
      const path = `/forecasts/search?${params}`;
      return this.fetchJson<SearchResponse>(path, { signal: options?.signal });
    }
    ```
    - NOT deduplicated (search terms change rapidly).
    - NOT circuit-broken (user expects immediate search feedback).
    - The `signal` parameter is critical for AbortController-based race condition prevention.
    - Import `SearchResponse` from types.

    **1c. Create SearchBar component:**
    Create `frontend/src/components/SearchBar.ts`:
    - Class `SearchBar` (does NOT extend Panel -- it's a lightweight inline component, not a full panel).
    - Constructor: creates the DOM structure using `h()`.
    - DOM structure: a single row containing:
      - Text input (`<input type="text" placeholder="Search forecasts...">`) with class `search-input`.
      - Country dropdown (`<select>`) with class `search-country-select`. Options: "All Countries" (value="") + populated dynamically from countries data. Initially just "All Countries".
      - Category dropdown (`<select>`) with class `search-category-select`. Options: "All Categories" (value=""), "Conflict", "Cooperation", "Diplomacy", "Economic", "Military".
    - Container class: `search-bar`.

    **Debounce mechanism:**
    - Implement a `debounce()` utility function (~8 lines) inside the file (or import if a utils/debounce.ts exists, but don't create a separate file for 8 lines).
    - Attach debounced (300ms) `input` event listener to the text input.
    - Attach immediate `change` event listener to both dropdowns.

    **Search execution:**
    - Private `searchController: AbortController | null` field.
    - `doSearch()` method:
      1. Abort previous in-flight request (`this.searchController?.abort()`).
      2. Create new AbortController.
      3. Read current values: query from input, country from dropdown, category from dropdown.
      4. If query is empty AND no country/category filter: dispatch `search-cleared` CustomEvent (ForecastPanel reverts to showing all forecasts).
      5. Otherwise: call `forecastClient.search(query, { country, category, signal })`.
      6. On success: dispatch `search-results` CustomEvent with the `SearchResponse` data.
      7. On AbortError: silently ignore (expected when new search supersedes old).
      8. On other error: dispatch `search-error` CustomEvent.

    **Public API:**
    - `getElement(): HTMLElement` -- returns the container.
    - `updateCountries(countries: CountryRiskSummary[]): void` -- populates country dropdown from data.
    - `setCountry(iso: string | null): void` -- programmatically sets country dropdown (for cross-column sync with Col 1).
    - `destroy(): void` -- removes event listeners, aborts pending request.

    **Export:** `SearchBar` class.
