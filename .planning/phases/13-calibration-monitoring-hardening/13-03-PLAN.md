  <name>Task 1: Create alert manager with SMTP sending and rate limiting</name>
  <files>src/monitoring/__init__.py, src/monitoring/alert_manager.py</files>
  <action>
  1. Create `src/monitoring/__init__.py` (empty or with `__all__` listing the modules).

  2. Create `src/monitoring/alert_manager.py`:

  `AlertManager` class:
  - `__init__(self, settings: Settings)` -- Reads smtp_host, smtp_port, smtp_username, smtp_password, smtp_sender, alert_recipient, alert_cooldown_minutes from settings. Initializes `_last_sent: dict[str, datetime]` for per-type cooldown tracking. If smtp_host is empty string, sets `self._enabled = False` and logs WARNING "SMTP not configured, alerts disabled".
  - `async send_alert(self, alert_type: str, subject: str, body: str) -> bool` -- Rate-limited alert sending:
    a. If not enabled, log and return False
    b. Check cooldown: if `_last_sent[alert_type]` exists and (now - last) < cooldown, log "suppressed" at DEBUG and return False
    c. Use `asyncio.to_thread(self._send_sync, subject, body)` to avoid blocking the event loop
    d. On success: update `_last_sent[alert_type]` and return True
    e. On failure: log ERROR with exception details, return False (NEVER let SMTP failure propagate -- fire-and-forget)
  - `_send_sync(self, subject: str, body: str) -> None` -- Synchronous SMTP send:
    a. Construct MIMEMultipart("alternative") with plain text body
    b. Subject: `[Geopol] {subject}`
    c. Connect with `smtplib.SMTP(host, port, timeout=10)`, starttls(), login(), send_message()
    d. Raise on failure (caught by caller)
  - `is_enabled: bool` property
  - `reset_cooldown(self, alert_type: str) -> None` -- For testing or manual override.

  Import: smtplib, email.mime.text, email.mime.multipart, asyncio, logging, datetime.
  </action>
  <verify>`uv run python -c "from src.monitoring.alert_manager import AlertManager; from src.settings import Settings; am = AlertManager(Settings()); print(f'enabled={am.is_enabled}'); print('OK')"`</verify>
  <done>AlertManager sends SMTP alerts with per-type cooldown, never blocks async event loop, never propagates SMTP failures</done>

<task type="auto">
  <name>Task 2: Create feed, drift, budget, and disk monitors</name>
  <files>src/monitoring/feed_monitor.py, src/monitoring/drift_monitor.py, src/monitoring/budget_monitor.py, src/monitoring/disk_monitor.py</files>
  <action>
  Create four monitoring modules:

  **feed_monitor.py** -- `FeedMonitor` class:
  - `__init__(self, async_session_factory, settings: Settings)` -- Reads feed_staleness_hours.
  - `async check_feed_freshness(self) -> dict` -- Queries most recent IngestRun with status="success" and daemon_type="gdelt". Returns dict with: `stale: bool`, `hours_since_last: float | None`, `last_run_at: datetime | None`, `threshold_hours: float`. Stale if hours_since_last > feed_staleness_hours or no successful runs exist.
  - `async check_and_alert(self, alert_manager: AlertManager) -> dict` -- Calls check_feed_freshness(). If stale, calls `alert_manager.send_alert("feed_stale", "GDELT Feed Stale", f"No new data for {hours:.1f} hours (threshold: {threshold}h)")`. Returns the freshness dict.

  **drift_monitor.py** -- `DriftMonitor` class:
  - `__init__(self, async_session_factory, settings: Settings)` -- Reads drift_threshold_pct. This replaces the legacy DriftDetector (JSON file storage) with PostgreSQL-backed rolling Brier score.
  - `async compute_rolling_brier(self, window_days: int = 30) -> dict` -- Queries predictions joined with outcome_records where outcome_records.resolution_date >= (now - window_days). Computes Brier score = mean((predicted_prob - outcome)^2). Returns: `brier_score: float | None`, `sample_count: int`, `window_days: int`, `sufficient_data: bool` (True if sample_count >= 20).
  - `async compute_baseline_brier(self) -> float | None` -- Computes all-time Brier score from all resolved predictions.
  - `async check_drift(self) -> dict` -- Computes rolling and baseline. If both available and sufficient data: drift_detected if rolling is > (1 + threshold_pct/100) * baseline. Returns: `drift_detected: bool`, `rolling_brier: float | None`, `baseline_brier: float | None`, `degradation_pct: float | None`, `sufficient_data: bool`.
  - `async check_and_alert(self, alert_manager: AlertManager) -> dict` -- Calls check_drift(). If drift_detected, sends alert.

  **budget_monitor.py** -- `BudgetMonitor` class:
  - `__init__(self, async_session_factory, settings: Settings)` -- Reads gemini_daily_budget.
  - `async get_budget_status(self) -> dict` -- Queries count of today's predictions (same logic as BudgetTracker._check_budget_pg but returns a status dict). Returns: `budget_total: int`, `budget_used: int`, `budget_remaining: int`, `pct_used: float`, `exhausted: bool`.
  - This is a read-only reporter, NOT the budget enforcer (that's BudgetTracker). It provides data for the health endpoint and daily digest.

  **disk_monitor.py** -- `DiskMonitor` class:
  - `__init__(self, settings: Settings)` -- Reads disk_warning_pct, disk_critical_pct, gdelt_db_path, log_dir.
  - `check_disk(self) -> dict` -- Uses `psutil.disk_usage("/")` (root partition). Returns: `percent_used: float`, `free_gb: float`, `total_gb: float`, `status: str` ("ok" | "warning" | "critical"), `warning_threshold: float`, `critical_threshold: float`.
  - `async emergency_cleanup(self) -> dict` -- Triggered when disk >= critical threshold. Ordered cleanup:
    a. Purge log files older than retention_days/2 from log_dir
    b. Purge GDELT data files beyond 90-day retention (if any exist in data/ directory)
    c. Return: `files_removed: int`, `bytes_freed: int`
  - `async check_and_alert(self, alert_manager: AlertManager) -> dict` -- Calls check_disk(). At warning: send alert. At critical: send alert AND run emergency_cleanup.

  All monitors are designed to be called from the daily pipeline or a periodic health check loop. They do NOT run background threads themselves.

  `import psutil` is needed for disk_monitor.py only. Add `psutil` to dependencies if not already present.
  </action>
