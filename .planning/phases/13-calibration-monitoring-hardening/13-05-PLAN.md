  2. Create `src/polymarket/client.py`:

  `PolymarketClient` class:
  - `GAMMA_API_BASE = "https://gamma-api.polymarket.com"`
  - `GEO_KEYWORDS = ["politic", "geopolitic", "world", "war", "election", "international", "government", "military", "conflict", "sanction", "nuclear", "nato", "diplomacy"]`
  - `__init__(self, session: aiohttp.ClientSession | None = None)` -- Optionally accepts a shared aiohttp session. If None, creates one internally. Tracks `_consecutive_failures: int = 0` and `_circuit_open: bool = False` for circuit breaker.
  - `async fetch_geopolitical_markets(self, limit: int = 100) -> list[dict]`:
    a. If circuit breaker open and fewer than 5 minutes since last failure, return empty list
    b. Discover tags via `GET /tags`. Filter tags whose label contains any GEO_KEYWORDS (case-insensitive)
    c. For each matching tag, fetch events via `GET /events?tag_id={id}&active=true&closed=false&limit={limit}`
    d. Deduplicate events by event ID
    e. On success: reset _consecutive_failures, return unique events
    f. On failure (aiohttp.ClientError, timeout, non-200): increment _consecutive_failures. After 5 consecutive failures, set _circuit_open=True. Log WARNING. Return empty list.
    g. Use `tenacity.retry` with 3 attempts, exponential backoff starting at 2s for individual HTTP requests (NOT the overall function)
  - `async fetch_event_prices(self, event_id: str) -> list[dict]`:
    Fetch current market prices for a specific event. Returns list of market dicts with outcomePrices. Used for hourly snapshots.
  - `async close(self)` -- Close internally-created session if applicable.

  Use existing `aiohttp` (already in project). Set request timeout to 15 seconds. Parse response defensively -- if response shape is unexpected (no "id", no "markets"), log WARNING and skip that event rather than crashing.

  DO NOT use the polymarket-apis SDK -- raw aiohttp is more reliable and avoids a new dependency.
  </action>

<task type="auto">
  <name>Task 2: Create matcher and comparison service</name>
  <files>src/polymarket/matcher.py, src/polymarket/comparison.py</files>
  <action>
  **matcher.py** -- `PolymarketMatcher` class:
  - `__init__(self, gemini_client, match_threshold: float = 0.6)` -- Takes the existing Gemini client (from ReasoningOrchestrator or GeminiClient wrapper). The match_threshold is from settings.polymarket_match_threshold.
  - `async match_event_to_predictions(self, polymarket_event: dict, active_predictions: list[Prediction]) -> tuple[str | None, float]`:
    Phase 1 -- Keyword pre-filter:
    a. Extract title and description from polymarket_event
    b. For each active prediction, compute: country_match (country_iso appears in PM text) OR keyword_overlap (>15% word overlap between prediction.question and PM title+description)
    c. Keep candidates passing either check, limit to top 10

    Phase 2 -- LLM ranking (only if candidates exist):
    a. Build prompt listing the PM question and candidates with their IDs and questions
    b. Ask Gemini to return JSON: {"match_id": "<id or null>", "confidence": 0.0-1.0}
    c. Use `asyncio.to_thread()` for the synchronous Gemini call
    d. Parse response JSON. If confidence >= match_threshold, return (prediction_id, confidence)
    e. On any LLM failure (parsing error, API error): log WARNING, return (None, 0.0)

    If no candidates pass pre-filter, return (None, 0.0) without calling LLM.

  **comparison.py** -- `PolymarketComparisonService` class:
  - `__init__(self, async_session_factory, polymarket_client: PolymarketClient, matcher: PolymarketMatcher, settings: Settings)`
  - `async run_matching_cycle(self) -> dict`:
    a. Fetch geopolitical markets via client
    b. Load active predictions from PostgreSQL (unexpired, with probability)
    c. For each market not already in polymarket_comparisons: run matcher
    d. For matches above threshold: create PolymarketComparison row (status="active")
    e. Return: `{"markets_fetched": N, "new_matches": M, "already_tracked": K}`
  - `async capture_snapshots(self) -> int`:
    a. Load all active PolymarketComparison rows
    b. For each: fetch current prices via client.fetch_event_prices()
    c. Create PolymarketSnapshot row with current polymarket_price and geopol_probability (re-query prediction for latest prob)
    d. Update last_snapshot_at on the comparison row
    e. Return count of snapshots captured
  - `async resolve_completed(self) -> int`:
    a. Load active comparisons where Polymarket event has resolved (need to check via client)
    b. For each resolved: compute Brier scores for both Polymarket and Geopol
    c. Geopol Brier: (geopol_probability - actual_outcome)^2
    d. Polymarket Brier: (polymarket_price_at_resolution - actual_outcome)^2
    e. Update comparison: status="resolved", polymarket_outcome, geopol_brier, polymarket_brier, resolved_at
    f. Return count of resolved comparisons
  - `async get_active_comparisons(self) -> list[dict]`:
    Query active comparisons with latest snapshot data. Return list of dicts ready for API response.
  - `async get_resolved_comparisons(self, limit: int = 50) -> list[dict]`:
    Query resolved comparisons ordered by resolved_at desc. Return list with Brier scores.
  - `async get_comparison_summary(self) -> dict`:
    Return summary stats: active_count, resolved_count, geopol_avg_brier, polymarket_avg_brier, geopol_wins (count where geopol_brier < polymarket_brier).
  </action>
