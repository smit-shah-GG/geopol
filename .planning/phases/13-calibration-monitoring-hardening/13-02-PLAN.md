<task type="auto">
  <name>Task 2: Create weight optimizer with L-BFGS-B Brier score minimization</name>
  <files>src/calibration/weight_optimizer.py</files>
  <action>
  Create `src/calibration/weight_optimizer.py` with:

  1. `optimize_alpha_for_category(outcomes: list[float], tkg_probs: list[float], llm_probs: list[float]) -> tuple[float, float]` -- Minimizes Brier score using `scipy.optimize.minimize(method='L-BFGS-B')` with alpha bounds [0.0, 1.0]. Returns (optimal_alpha, achieved_brier_score). Follows the pattern from temperature_scaler.py. Use x0=[0.6], maxiter=200, ftol=1e-10.

  2. `CalibrationResult` dataclass:
     - `new_weights: dict[str, tuple[float, float, int]]` -- cameo_code -> (alpha, brier, sample_size)
     - `flagged_weights: dict[str, str]` -- cameo_code -> flag_reason
     - `applied_weights: dict[str, tuple[float, float, int]]` -- weights that were auto-applied
     - `held_weights: dict[str, tuple[float, float, int]]` -- weights held for manual review
     - `timestamp: datetime`

  3. `WeightOptimizer` class:
     - `__init__(self, async_session_factory, settings: Settings)` -- stores session factory and reads calibration_min_samples, calibration_max_deviation from settings
     - `async run_weekly_calibration() -> CalibrationResult` -- full pipeline:
       a. Query all resolved prediction-outcome pairs by joining `predictions` and `outcome_records` tables. Extract `llm_probability` and `tkg_probability` from `ensemble_info_json` blob, and `cameo_root_code` from predictions.
       b. Group by CAMEO root code (skip predictions with no cameo_root_code)
       c. For each group with >= min_samples: call `optimize_alpha_for_category()`
       d. Aggregate under-sampled codes to super-categories (using CAMEO_TO_SUPER) and optimize those
       e. Compute global alpha from ALL resolved pairs (if >= min_samples)
       f. Apply guardrails: compare each new weight against current CalibrationWeight row. If relative deviation > max_deviation (20%), flag the weight.
       g. Auto-apply unflagged weights to `calibration_weights` table (upsert by cameo_code)
       h. Persist ALL weights (flagged and unflagged) to `calibration_weight_history`
       i. Return CalibrationResult

     - `async _fetch_resolved_pairs(session) -> list[dict]` -- Queries predictions joined with outcome_records. Each dict has: prediction_id, cameo_root_code, llm_probability, tkg_probability, outcome, category.
     - `_check_guardrails(new_weights, current_weights, max_deviation) -> dict[str, str]` -- Returns flagged weights with reason strings.

  Use `from src.calibration.priors import CAMEO_TO_SUPER, SUPER_CATEGORIES, COLD_START_PRIORS` for the hierarchy.

  Error handling: If L-BFGS-B fails for a category, log WARNING and skip that category (don't crash the entire calibration run). If no resolved pairs exist at all, return empty CalibrationResult with a log message.

  DO NOT import or use AlertManager here -- that coupling happens in Plan 06 (pipeline hooks). Keep this module pure optimization logic.
  </action>
  <verify>`uv run python -c "from src.calibration.weight_optimizer import optimize_alpha_for_category, WeightOptimizer, CalibrationResult; alpha, brier = optimize_alpha_for_category([1.0, 0.0, 1.0], [0.8, 0.3, 0.7], [0.9, 0.1, 0.8]); print(f'alpha={alpha:.3f}, brier={brier:.4f}'); assert 0.0 <= alpha <= 1.0; print('OK')"`</verify>
  <done>Weight optimizer produces valid alpha values from test data; CalibrationResult correctly partitions flagged vs applied weights</done>
</task>

<task type="auto">
  <name>Task 3: Create weight loader with hierarchical fallback and caching</name>
  <files>src/calibration/weight_loader.py</files>
  <action>
  Create `src/calibration/weight_loader.py` with:

  1. `WeightLoader` class:
     - `__init__(self, async_session_factory, min_samples: int = 10, cache_ttl_seconds: int = 300)` -- TTL-based in-memory cache to avoid hitting PostgreSQL on every prediction.
     - `async load_weights(self) -> None` -- Loads all CalibrationWeight rows from PostgreSQL into `self._weights: dict[str, CalibrationWeight]` keyed by cameo_code. Also loads COLD_START_PRIORS as fallback defaults. Sets `self._loaded_at`.
     - `async resolve_alpha(self, cameo_root_code: str | None = None, keyword_category: str | None = None) -> float` -- Hierarchical resolution:
       a. If cache expired (> cache_ttl_seconds since _loaded_at), reload weights
       b. If cameo_root_code provided and weights exist for it with sample_size >= min_samples, return that alpha
       c. Derive super-category via `infer_super_category(cameo_root_code, keyword_category)`. If weights exist for `"super:{super_cat}"` with sample_size >= min_samples, return that alpha
       d. If weights exist for `"global"` with sample_size >= min_samples, return that alpha
       e. Fall back to COLD_START_PRIORS: try super-category prior, then global prior (0.58)
     - `async get_weight_info(self, cameo_root_code: str | None = None, keyword_category: str | None = None) -> dict` -- Returns dict with resolved alpha, resolution_level ("cameo"|"super"|"global"|"prior"), and sample_size for logging/debugging.
     - Property `weights_loaded: bool` -- True if weights have been loaded at least once.

  Import from `src.calibration.priors`: CAMEO_TO_SUPER, COLD_START_PRIORS, infer_super_category.
  Import from `src.db.models`: CalibrationWeight.

  The loader is designed to be instantiated once and reused across predictions (singleton in the API server, created at startup). The cache prevents per-request PostgreSQL queries while allowing the weekly calibration to take effect within 5 minutes.
  </action>
  <verify>`uv run python -c "from src.calibration.weight_loader import WeightLoader; print('WeightLoader imported OK')"`</verify>
  <done>WeightLoader provides hierarchical alpha resolution with TTL-cached weights, falling back through CAMEO -> super-category -> global -> cold-start priors</done>
