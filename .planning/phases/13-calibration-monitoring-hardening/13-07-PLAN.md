  **calibration.py** -- New API route module:

  1. `GET /api/v1/calibration/polymarket` -- Returns Polymarket comparison data:
     - Response schema: `{"active": [...], "resolved": [...], "summary": {...}, "seeking_more_matches": bool}`
     - `active`: list of active comparisons with polymarket_title, polymarket_price, geopol_probability, match_confidence, last_snapshot_at
     - `resolved`: list of resolved comparisons with Brier scores for both sources
     - `summary`: active_count, resolved_count, geopol_avg_brier, polymarket_avg_brier, geopol_wins
     - `seeking_more_matches`: True if active_count < 5
     - Uses PolymarketComparisonService.get_active_comparisons(), get_resolved_comparisons(limit=20), get_comparison_summary()
     - Requires API key auth (like other data endpoints)

  2. `GET /api/v1/calibration/weights` -- Returns current calibration weights:
     - Response: list of dicts with cameo_code, alpha, sample_size, brier_score, updated_at
     - Query CalibrationWeight table
     - Requires API key auth

  3. `GET /api/v1/calibration/weights/history` -- Returns weight version history:
     - Query params: `limit` (default 50), `cameo_code` (optional filter)
     - Response: list of CalibrationWeightHistory rows
     - Requires API key auth
