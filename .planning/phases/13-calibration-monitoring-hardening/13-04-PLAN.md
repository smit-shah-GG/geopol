<task type="auto">
  <name>Task 2: Create systemd service and timer units</name>
  <files>systemd/geopol-ingest.service, systemd/geopol-rss.service, systemd/geopol-daily-forecast.service, systemd/geopol-daily-forecast.timer</files>
  <action>
  Create `systemd/` directory at project root and add 4 unit files:

  **systemd/geopol-ingest.service** -- GDELT micro-batch ingest daemon:
  ```ini
  [Unit]
  Description=Geopol GDELT Ingest Daemon
  After=network-online.target postgresql.service redis.service
  Wants=network-online.target

  [Service]
  Type=simple
  User=geopol
  WorkingDirectory=/opt/geopol
  ExecStart=/opt/geopol/.venv/bin/python -m src.pipeline.ingest_daemon
  Restart=on-failure
  RestartSec=30
  Environment=PYTHONUNBUFFERED=1
  EnvironmentFile=/opt/geopol/.env

  [Install]
  WantedBy=multi-user.target
  ```

  **systemd/geopol-rss.service** -- RSS feed poller daemon:
  ```ini
  [Unit]
  Description=Geopol RSS Feed Poller
  After=network-online.target postgresql.service
  Wants=network-online.target

  [Service]
  Type=simple
  User=geopol
  WorkingDirectory=/opt/geopol
  ExecStart=/opt/geopol/.venv/bin/python -m src.pipeline.rss_daemon
  Restart=on-failure
  RestartSec=60
  Environment=PYTHONUNBUFFERED=1
  EnvironmentFile=/opt/geopol/.env

  [Install]
  WantedBy=multi-user.target
  ```

  **systemd/geopol-daily-forecast.service** -- Daily forecast pipeline (triggered by timer):
  ```ini
  [Unit]
  Description=Geopol Daily Forecast Pipeline
  After=network-online.target postgresql.service

  [Service]
  Type=oneshot
  User=geopol
  WorkingDirectory=/opt/geopol
  ExecStart=/opt/geopol/.venv/bin/python -m src.pipeline.run_daily
  Environment=PYTHONUNBUFFERED=1
  EnvironmentFile=/opt/geopol/.env
  TimeoutStartSec=3600
  ```

  **systemd/geopol-daily-forecast.timer** -- Triggers daily forecast at 06:00 UTC:
  ```ini
  [Unit]
  Description=Geopol Daily Forecast Timer

  [Timer]
  OnCalendar=*-*-* 06:00:00 UTC
  Persistent=true
  RandomizedDelaySec=300

  [Install]
  WantedBy=timers.target
  ```

  Add a brief README comment at the top of each file noting: "Template service file. Edit paths for your deployment. Copy to /etc/systemd/system/ and enable."

  These are reference templates -- the user will adapt paths (/opt/geopol, .venv location) for their deployment. The key behavioral properties are: Restart=on-failure for daemons, Type=oneshot for the pipeline, and Persistent=true for the timer (catches up missed runs).
  </action>
  <verify>`ls /home/kondraki/personal/geopol/systemd/ | wc -l` shows 4 files; each file contains the expected directives</verify>
  <done>Four systemd unit files with Restart=on-failure for daemons, daily timer at 06:00 UTC, all configurable via .env</done>
