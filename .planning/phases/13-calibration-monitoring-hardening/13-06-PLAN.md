       a. Call `asyncio.run(self.weight_loader.resolve_alpha(cameo_root_code=..., keyword_category=category))` -- BUT since _combine_predictions is called synchronously from predict(), and predict() is called via asyncio.to_thread() from the daily pipeline, we need a synchronous path. Solution: add a `_cached_alpha: float | None` field set by a new `async def prepare_weights(self, cameo_root_code: str | None, keyword_category: str | None)` method that must be called BEFORE predict(). The daily pipeline will call this async method before calling predict() via to_thread().
       b. ALTERNATIVELY (simpler): Make `_combine_predictions()` accept an `alpha_override: float | None` parameter. The `predict()` method uses `self.alpha` as default. The daily pipeline passes the resolved alpha. This avoids async/sync mixing inside predict().
       c. USE THE SIMPLER APPROACH: Add `alpha_override: float | None = None` to `predict()` signature. When provided, use it instead of self.alpha in `_combine_predictions()`. Log the resolved alpha and its resolution level (cameo/super/global/prior).

     - Update `weights_used` tuple in EnsemblePrediction to reflect the dynamic alpha

  IMPORTANT: The existing predict() method signature must remain backward compatible. All new parameters are Optional with defaults.
